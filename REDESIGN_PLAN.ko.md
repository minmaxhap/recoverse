# Recoverse 재설계 계획

## 목표

Recoverse를 **Story Book x Time Capsule** 모바일 경험으로 전환한다. 이번 전환의 목적은 기능 확장이 아니라 제품 정체성, 화면 구조, 디자인 토큰을 먼저 일관되게 고정하는 것이다.

## 잠긴 데이터 경계

아래 계약은 재설계 중에도 유지한다.

| 경계 | 계약 |
| --- | --- |
| 로컬 저장 | `recoverse_reflections_v1` 키와 `Reflection[]` 구조 |
| 백업/가져오기 | `recoverse_reflections_v1` 백업 스키마와 병합 규칙 |
| 공유 링크 | `#share=` URL 해시와 `recoverse_shared_reflection_v1` 스냅샷 |
| 설정 | `recoverse_language`, `recoverse_theme` |
| 계정 저장 준비 | `recoverse_account_sync_v1` 페이로드 |

## 새 설계 원칙

- 코드 변경보다 디자인 시스템과 흐름 정렬을 먼저 끝낸다.
- 새 의존성은 실제 병목이 생길 때만 추가한다.
- 우주 메타포는 기본 제품 언어에서 제거한다.
- 모든 색, 여백, 타입, 모션은 [DESIGN.md](./DESIGN.md)에 연결한다.
- 모바일 390 x 844 기준에서 먼저 맞춘다.
- 구현 후 375 / 768 / 1280px 시각 QA를 한다.

## 화면 전환 계획

| 단계 | 대상 | 결과 |
| --- | --- | --- |
| 1 | 전역 CSS | `book-capsule` 토큰과 기본 표면 정리 |
| 2 | Home | 30초 CTA, 샘플 회고 카드, 앨범 진입 |
| 3 | 회고 시작 | 30초/연말/여행/친구 비교 모드 카드 |
| 4 | 질문 작성 | 질문 카드와 편지지 입력 |
| 5 | 작성 완료 | 왁스 실링 완료 화면 |
| 6 | 회고 앨범/상세 | 책장형 목록과 편지형 상세 |
| 7 | 연말/여행/친구 비교 | 새 화면 골격과 기존 데이터 연결 |
| 8 | 설정/디자인 시스템 | 앱 내부 시스템 확인 화면 |

## 구현 메모

- 현재 `HomeUniverseView`는 이름을 당장 바꾸지 않아도 되지만, 내부 메타포는 Home/Album으로 교체한다.
- `ReviewAgainPage`는 회고 앨범과 친구 비교로 분리할 수 있다.
- `NewReflectionPage`의 선택 UI는 회고 시작 화면으로 단순화한다.
- `WriteReflectionPage`는 편지지 입력과 seal CTA를 중심으로 다시 설계한다.
- 완료 전용 `reflection-complete` 모드 추가를 고려한다.
- `ArchiveSettingsTools`는 설정 화면으로 유지하되 시각 언어를 교체한다.

## 검증 기준

- 모든 요청 화면 12개가 코드 또는 명확한 화면 골격으로 존재한다.
- 모바일 화면에서 텍스트가 버튼/카드 밖으로 밀리지 않는다.
- 한글 제목이 어색하게 한 글자씩 떨어지지 않는다.
- 주요 버튼은 44px 이상 터치 영역을 가진다.
- 저장/백업/공유 테스트가 계속 통과한다.
