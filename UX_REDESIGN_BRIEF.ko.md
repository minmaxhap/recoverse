# Recoverse UX 재설계 브리프

## 1. 진단

현재 앱은 회고 데이터와 저장 흐름은 갖추고 있지만, 시각 언어가 제품 감정과 맞지 않는다.

| 영역 | 문제 |
| --- | --- |
| 홈 | 우주/갤럭시 메타포가 강해 회고가 “보관된 편지”가 아니라 장식적 대시보드처럼 보인다. |
| 작성 진입 | 빠른 작성보다 선택/설명 구조가 먼저 보인다. |
| 작성 | 질문에 답하는 경험보다 UI 단계를 통과하는 느낌이 강하다. |
| 완료 | 저장 완료가 브랜드 순간으로 기억되지 않는다. |
| 다시 보기 | 회고를 다시 여는 감정보다 목록/기능 탐색 느낌이 강하다. |
| 설정 | 백업/언어/테마 기능은 있지만 새 제품 정서와 연결되어 있지 않다. |

## 2. 새 UX 방향

새 UX는 **Story Book x Time Capsule**을 기준으로 한다.

- 사용자는 30초 안에 회고를 시작한다.
- 질문 작성은 편지지처럼 느껴진다.
- 완료는 왁스 실링으로 봉인되는 순간처럼 보인다.
- 다시 보기는 앨범과 책장처럼 구성된다.
- 연말/여행/친구 비교는 같은 회고 시스템의 확장이다.

## 3. 핵심 화면별 요구사항

### Splash

- Recoverse 브랜드와 책/편지 감성만 남긴다.
- 긴 설명을 넣지 않는다.

### Home

- 첫 문장은 “30초면 충분해요.”로 시작한다.
- 주 CTA는 `30초 만에 시작하기`.
- 보조 CTA는 `예전 회고 보기`.
- 샘플 회고 카드는 세 장 이하로 노출한다.

### 회고 시작

- 30초 회고를 기본 선택으로 둔다.
- 연말, 여행, 친구 비교는 같은 화면의 카드로 진입한다.
- 복잡한 기간 설정은 첫 화면에 노출하지 않는다.

### 질문 작성

- 질문은 한 번에 하나씩 보여준다.
- 입력 영역은 편지지처럼 보여야 한다.
- 글자 수, 자동 저장, 감정 칩은 보조 정보로만 둔다.

### 작성 완료

- 왁스 실링 또는 봉투 봉인 모션이 중심이다.
- `회고가 봉인됐어요.` 계열 문구를 사용한다.
- 다음 행동은 앨범 보관과 하나 더 쓰기 정도로 제한한다.

### 회고 앨범

- 월, 여행, 친구, 연말 묶음을 책장처럼 보여준다.
- 카드 썸네일은 사진이 아니라 감정 표지 역할을 한다.

### 회고 상세

- 편지 한 장처럼 읽히는 레이아웃.
- 본문 텍스트가 가장 중요하다.
- 태그와 사진 조각은 본문보다 조용해야 한다.

### 연말 회고

- 통계는 보조다.
- 올해 남은 문장, 자주 나온 감정, 월별 하이라이트를 중심으로 한다.

### 여행 회고

- 지도는 정확한 내비게이션이 아니라 엽서다.
- 장소와 감정의 연결을 시각화한다.

### 친구 비교

- 비교는 점수화가 아니다.
- 같은 질문을 서로 다른 답변으로 읽는 경험이어야 한다.

### 설정

- 설정 행도 종이 카드 언어를 따른다.
- 알림, 잠금, 테마, 내보내기/가져오기를 우선한다.

### 디자인 시스템

- 앱 내부에서 토큰, 버튼, 카드, 아이콘 샘플을 확인하는 화면이다.
- 향후 디자인 QA 기준으로 쓴다.

## 4. 문구 원칙

사용:

- 30초면 충분해요.
- 회고가 봉인됐어요.
- 나의 책고
- 회고 앨범
- 예전 회고 보기
- 오늘의 질문
- 같은 질문, 다른 기억

피함:

- 기능을 설명하는 긴 문장
- 대시보드, 분석, 통계 중심 문구
- 관측, 행성, 은하, 궤도 같은 우주 메타포
- 좋아요, 댓글, 피드 중심 SNS 문구

## 5. 구현 순서

1. 디자인 토큰을 CSS 변수로 정리한다.
2. Home을 새 디자인으로 교체한다.
3. 회고 시작/질문 작성/작성 완료 흐름을 바꾼다.
4. 다시 보기를 회고 앨범/상세로 재구성한다.
5. 연말/여행/친구 비교 화면 골격을 추가한다.
6. 설정과 디자인 시스템 화면을 추가한다.
7. 브라우저 시각 QA로 overflow, CJK 줄바꿈, 터치 영역을 확인한다.

## 6. 변경하지 않는 경계

- `Reflection` 타입과 저장 스키마
- `recoverse_reflections_v1` 백업 호환성
- URL 해시 기반 공유 스냅샷
- 계정 저장 계약
- 테스트 가능한 저장/백업/공유 유틸리티
