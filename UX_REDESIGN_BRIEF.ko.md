# Recoverse UX 재설계 브리프

## 1. 진단: 현재 디자인의 문제

| 영역 | 문제 |
| --- | --- |
| 홈 | "저널" 메타포 (따뜻한 양피지, 세리프, 기울기)가 "밤하늘" 아이덴티티와 충돌. `cosmicWash`는 장식용 그라데이션일 뿐 구조적이지 않음. 빈 상태 문장이 너무 김. |
| 작성 진입 | 4단계 위자드(유형 → 기간 → 깊이 → 검토)는 사용 전에 설명을 3번 읽어야 함. `"처음에는 여행부터…"`는 UX가 아니라 핑계. |
| 작성 | 질문 카드 배경 `#f5f0e8`로 인해 낮-밤 대비가 깨짐. 하단 액션 바가 3열로 복잡. |
| 시각 정체성 | CSS 변수(`--color-page: #0B0F1E`)가 디자인 시스템에서 정한 `space.black: #08070F`와 다름. 금색은 많고 보라/라벤더는 거의 안 보임. 밤하늘 느낌보다 금색 강조 UI에 가까움. |
| 문구 | 설명형 문장이 많음. "짧은 질문에 답하면~" → 앱이 UX를 문장으로 설명하고 있음. |

## 2. 방향: LazyCodex 6원칙

### 2.1 조용한 발견 (Quiet Discovery)

사용자에게 기능을 설명하지 않는다. 앱이 "하고 싶은 일"을 미리 말하지 않고, 사용자가 직접 발견하게 둔다.

- **지금**: 홈에 `"짧은 질문에 답해두면, 시간이 지난 뒤…"` 설명 문단이 있음.
- **목표**: 빈 상태는 첫 행동 CTA 하나 + 배경 장면만. 설명은 하단 탭 아이콘 레이블과 문맥에 맡긴다.

### 2.2 0초 진입 (Zero-Second Entry)

앱을 열자마자 바로 쓸 수 있어야 한다. 설정/선택/위자드는 쓰기 시작한 뒤에 둔다.

- **지금**: 새 회고를 시작하려면 유형(4개 중 고름) → 기간(입력) → 깊이(4개 중 고름) → 검토 후에야 질문 카드가 나타남.
- **목표**: 하단 `기억 작성` 탭을 누르면 바로 첫 질문이 보인다. 유형/기간 선택은 필요한 순간(저장 전)에 묻는다.

### 2.3 질문 우선 작성 (Question-First Writing)

위자드를 없애고, 사용자는 "질문"을 먼저 마주한다.

- **현재 위자드 4단계를 없앰**: 유형? → "여행". 기간? → "제주 여행". 깊이? → "가볍게". 검토? → "시작".
- **바뀔 흐름**: `기억 작성 탭` → `"무엇을 돌아볼까요?"` (한 줄 입력 + 추천칩) → `첫 질문 카드`.
- 질문 카드가 앱의 첫 작성 화면이어야 한다. 위자드가 먼저 나오면 안 됨.

### 2.4 더 단순한 홈 (Simpler Home)

현재 홈은 "장면"을 만들기 위해 CSS 그라데이션 4겹, `::before`/`::after`, journalSheet 회전, 떠다니는 조각을 모두 사용한다. 하지만 실제 데이터 표현력은 약함.

- **지금**: 하나의 `featuredReflection` 카드 + 최대 4개의 떠다니는 레이블. 대부분의 화면은 장식.
- **목표**: 연도별 기억 묶음이 중심. 기억 오브젝트는 별/궤도로 표현하고, 선택 시 미리보기 패널을 연다. 데이터가 많을수록 화면이 풍성해져야 함.

### 2.5 더 단순한 다시 보기 (Simpler Review)

같은 질문 비교, 공유, 상세 읽기가 뒤섞여 있다.

- **지금**: 상세 하단에 `[같은 질문 비교]`와 `[공유하기]` 버튼이 함께 있음.
- **목표**: 다시 보기 탭은 목록 → 상세 읽기. 비교와 공유는 상세 화면에서 접근할 수 있지만 같은 우선순위가 아님.

### 2.6 응집된 밤하늘 시각 아이덴티티 (Cohesive Night-Sky Visual Identity)

현재 디자인 시스템에 정의된 색상(`space.black #08070F`, `space.deepNight #111322`, `Star Lavender #B8A6E8`, `Orbit Purple #6E5A9A`, `Memory Gold #F4C56A`)을 정확히 구현한다.

- **지금**: 배경 `#0B0F1E` (매우 어두운 네이비). 표면 `#141C2E`. 금색 CTA. 따뜻한 저널 양피지가 우주 테마와 충돌.
- **목표**:
  - 배경: `#08070F` (완전한 검은 밤)
  - 표면: `#111322` (깊은 밤), `#1A2133` (부드러운 네이비)
  - 강조: 금색은 중요한 CTA와 대표 기억에만 사용
  - 보라/라벤더를 구조적 색상으로 사용 (궤도, 별, 선택 상태)
  - 모든 표면이 우주 테마 아래 통일됨. 따뜻한 저널 메타포 없음.

## 3. 구체적 변경 사항

### 3.1 홈 리라이트

- **제거**: `journalSheet`, `journalCopy`, `::before`/`::after` 양피지 효과, `cosmicWash` 장식 그라데이션, `driftLayer`/`driftMark`, `memoryFragment` (떠다니는 레이블), 세리프 폰트, 따뜻한 갈색 계열
- **추가**: 진짜 밤하늘 배경(`space.black`), 별 필드(작은 점들, CSS로 렌더링), 연도별 기억 묶음을 행성/별 오브젝트로 표현, 미리보기 패널(선택 시 아래/옆에서 열림)
- **빈 상태**: 짧은 헤드라인 + `[첫 기억 작성하기]` CTA 하나. 설명 문단 없음.
- **데이터 있음**: 기억 오브젝트가 배경에 흩어져 있음. 연도별 묶음. 각 오브젝트는 클릭 가능하며 미리보기 패널을 염.

### 3.2 작성 진입 리라이트

- **제거**: 4단계 위자드 (`NewReflectionPage.vue`의 템플릿 선택, 기간 입력, 깊이 선택, 검토 단계)
- **변경**: 하단 `기억 작성` 탭을 누르면 바로 질문 카드로 진입. 위자드 없음.
- **대체**: 저장 직전에(또는 작성 중) 간단히 주제/제목만 묻는 한 줄 입력. 한 화면, 반드시 입력 필요 없음 (기본값 자동 생성: "여행의 기억" 등).
- **깊이 선택 제거**: 기본 질문 수(5-7개)로 고정. "더 길게 쓰기"는 질문 카드 내 옵션으로 유지.

### 3.3 작성 페이지 리라이트

- **변경**: 질문 카드 배경을 `#f5f0e8`(밝은 낮)에서 `space.softNavy #1A2133`으로 변경
- **변경**: 하단 액션 바 단순화 (`[이전] [건너뛰기] [답변 저장]` → `[이전] [다음]`으로 간소화. 건너뛰기는 자동 처리)
- **추가**: 질문 카드 전환 시 부드러운 슬라이드 애니메이션 (DESIGN_SYSTEM 문서 준수)

### 3.4 다시 보기 리라이트

- **변경**: 목록을 현재 연도 중심으로 단순화. 복잡한 연도/주제 분류 대신 시간순 목록.
- **변경**: 상세 뷰에서 대표 장면/감정이 상단에 강조되어야 함. `[같은 질문 비교]`와 `[공유하기]`를 하단에 작게.

### 3.5 시각 아이덴티티 통일

| 토큰 | 현재 값 | 목표 값 |
| --- | --- | --- |
| 배경 | `--color-page: #0B0F1E` | `--color-page: #08070F` (space.black) |
| 홈 배경 | 그라데이션 + 양피지 | `--color-bg: #111322` (space.deepNight) |
| 표면 | `--color-surface: #141C2E` | `--color-surface: #1A2133` (space.softNavy) |
| 강조 CTA | `--color-gold: #F4C56A` | 유지하되 사용 범위 축소 |
| 보조 포인트 | 없음 | `--color-accent: #6E5A9A` (Orbit Purple) |
| 별 | 없음 | `--color-star: #B8A6E8` (Star Lavender) |
| 질문 카드 | `#f5f0e8` (밝음) | `--color-surface: #1A2133` (어두움, 통일) |

## 4. 실행 순서

| 단계 | 내용 |
| --- | --- |
| 1 | CSS 변수를 디자인 시스템 값과 정확히 일치시킴. 테마 통일. |
| 2 | 홈을 저널 메타포에서 밤하늘 오브젝트 필드로 전환. 빈 상태/데이터 상태 모두 재구성. |
| 3 | 작성 위자드 제거, 질문 우선 진입으로 전환. |
| 4 | 작성 카드 디자인을 어두운 표면으로 통일, 액션 바 단순화. |
| 5 | 다시 보기 뷰를 단순화, 시각 정체성 통일. |
| 6 | 문구 전체 audit: 설명 제거, 행동 중심으로. |

## 5. 금지 사항

- 데이터 스키마 변경 (`Reflection`, `recoverse_reflections_v1` 구조 유지)
- 새 npm 의존성 추가
- Three.js 또는 WebGL 도입
- 기능 범위 확장 (로그인, 클라우드, PDF 등은 이 브리프 범위 밖)
- `PRODUCT_SPEC`, `DESIGN_SYSTEM`, `USER_FLOW`의 방향과 충돌하는 결정
