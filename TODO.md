# Recoverse TODO

이 문서는 아직 구현되지 않았거나, 현재는 골격만 있는 기능을 실제 제품 흐름으로 완성하기 위한 작업 목록이다.

## 작업 규칙

- 한 번에 하나의 단계만 진행한다.
- 각 단계는 테스트 가능한 상태로 마감하고 커밋한다.
- 코드 변경 전에는 현재 UX 흐름과 데이터 경계를 먼저 확인한다.
- 기존 localStorage 데이터와 JSON import/export 호환성을 깨지 않는다.
- 결정이 필요한 제품 범위만 사용자에게 묻고, 구현 세부는 코드베이스 패턴에 맞춰 진행한다.

## 현재 완료된 기반

- 홈 우주 뷰와 행성 상세 화면 분리
- 전역 Topbar 제거
- 아카이브 IA를 `보관함 / 시간여행 / 정리 / 설정`으로 재구성
- JSON 관리, 언어 설정, 위험 작업을 아카이브 설정으로 이동
- 그룹 은하와 관측 모드의 화면 골격 및 타입 초안 추가
- 프로젝트 Markdown 문서 정리

목표: 실제 브라우저 리뷰에서 확인된 읽기 방해, 비활성 은하, 상세 대비, 모바일 밀도 문제를 우선 해결한다.

- [x] 모바일 관측 모드에서 행성 비주얼이 질문 카드를 덮지 않게 수정한다.
  - 확인: 모바일 공유/읽기 전용 화면에서 질문과 답변이 행성 그래픽에 가려지지 않는다.
- [x] 행성 상세 메타 카드의 밝은 회색/저대비 스타일을 다크 우주 테마에 맞춘다.
  - 확인: 유형, 설명, 최근 수정, 질문/답변 카드가 다크 화면 안에서 자연스럽게 읽힌다.
- [x] 홈 지도 안 비활성 그룹 은하 노드를 실제 생성 진입으로 연결하거나 명확한 상태로 바꾼다.
  - 확인: `그룹 은하`가 버튼처럼 보이면 클릭 시 생성 플로우로 이어진다.
- [x] 모바일 홈의 기억 지도 높이와 CTA 위치를 조정한다.
  - 확인: 작은 화면에서 새 기억 CTA와 하단 내비게이션 접근이 과도한 스크롤에 의존하지 않는다.
- [x] 행성 삭제 버튼을 상세 작성 흐름의 첫 행동에서 분리한다.
  - 확인: 삭제는 명시적으로 위험 영역을 펼쳤을 때만 보인다.
- [x] 주요 버튼/문구를 사용자의 행동 중심으로 정리한다.
  - 확인: 오브젝트, 관측 초대장, 공유된 탐사 로그처럼 모호한 문구가 줄어든다.

### 36단계: 제품 설계 명세서 v1.0 기반 Reflection 데이터 기반 추가

- [x] MVP 4개 템플릿(한 해, 여행, 시기, 나이 회고)과 질문 세트 모드를 추가한다.
  - 확인: 가볍게/깊게/공유용/비교용 질문 선택이 가능하다.
