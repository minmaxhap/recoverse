# Recoverse — 제품 스펙 (Codex 전달용)

## 0. 한 줄 정의

> 친구들과 함께 회고하는 시간을 한 권의 잡지처럼 남기고, 시간이 지나 다시 발견하는 인터랙티브 회고 플랫폼.

**핵심 철학**: 기록은 목적이 아니라 결과물이다. 진짜 가치는 (1) 함께하는 세션 경험과 (2) 시간이 지나 다시 발견하는 경험에 있다.

메타포: 세션 하나 = 잡지 한 호(Issue). 세션을 마치면 "발행"되어 책장에 꽂힌다. 연말호뿐 아니라 여행 특집호, 월호, 프로젝트 특집호로 확장된다.

**지켜야 할 원칙 5가지**
1. 회고를 쉽게 시작하게 만든다.
2. 혼자보다 함께하는 경험을 만든다.
3. 매년 새로운 회고 방식을 제안한다.
4. 자동으로 아름답게 편집한다.
5. 몇 년 뒤 다시 열어보고 싶은 서비스를 만든다.

## 1. 전체 기능 명세

### 1.1 회고 세션 (핵심 객체)
- **그룹 라이브 세션**: 각자 자기 폰으로 코드 합류 → 돌아가며 질문 → 전원 답 작성 → 동시 공개 → 발행
- **개인(솔로) 세션**: 혼자 질문을 던지고 답하는 발행 플로우. 연말/월간/여행/프로젝트/독서/자유 회고
- **복간(과거 회고 가져오기)**: 이미 작성한 회고를 책장에 넣는 세 가지 방식
  - **직접 옮겨 적기**: 질문·답 타이핑 (Phase 1)
  - **사진 스캔 보관**: 종이 회고를 찍으면 원본 그대로 "스캔 호"로 책장에 꽂힘. 구조화 없이도 보관 완료 — 잃어버림 방지가 1차 가치. 이후 "이 호를 글로 풀기" 버튼으로 **AI 전사**(사진 → 질문/답 초안 → 사용자 확인·수정) 시 다시 발견에 연결 (Phase 2)
  - **구글 폼/CSV 가져오기**: 구글 폼 응답을 CSV로 내보내 업로드 → 컬럼(질문)·행(응답자) 매핑 → 호로 변환 (Phase 3)

### 1.2 질문 시스템
- **질문 추천**: 상황별 질문 팩 — 연말 / 여행 / 커플 / 친구 / 졸업 / 퇴사 / 20대 인터뷰 / 부모님 인터뷰
- **질문 난이도**: 가볍게 / 적당히 / 깊게
- **친구 질문**: 세션에서 친구가 직접 작성 (Phase 1의 라운드제가 이것)
- **중복 방지**: 라운드제 + 이전 질문 노출로 구조적으로 해결

### 1.3 회고 포맷 (매년 새로운 방식)
올해를 건물로 / 영화 포스터로 / 올해의 세 장면 / 플레이리스트 / 지도 위에 표현 / 잡지 인터뷰 형식 / 서로 인터뷰하기 / 서로에게 편지 쓰기 / 올해의 키워드 / 올해의 색 / 미래의 나에게 편지

### 1.4 AI (진행자 역할)
- **질문 추천**: 세션 맥락에 맞는 질문 제안
- **생각 확장**: 답변 작성 중 후속 질문 — "왜 그렇게 느꼈어?", "그때 가장 기억나는 장면은?", "그 경험 이후 달라진 점은?"
- **포맷 추천**: 이번 회고에 어울리는 포맷 제안
- **변화 발견**: 반복되는 고민, 가장 많이 언급한 사람, 자주 간 장소, 성장한 점

### 1.5 자동 편집
세션이 끝나면 사용자가 디자인하지 않아도 잡지 / 책 / 인터뷰집 / 사진집 형태로 자동 편집.

### 1.6 책장
발행된 호가 책처럼 꽂힌다 — "2026 연말호", "제주 여행 특집호", "프로젝트 회고".

### 1.7 다시 발견 (제품의 존재 이유)
- **같은 질문 다른 해**: 2016 → 2019 → 2026 타임라인
- **친구 비교**: 같은 질문에 대한 친구들 답 나란히
- **올해 vs 작년**
- **오늘의 재발견**: 랜덤 추억
- **1년 전 오늘**
- **AI가 발견한 변화**

### 1.8 공유
읽기 전용 링크 / PDF / 인쇄 — 실제 잡지처럼 저장.

## 2. 단계별 로드맵 (구현 순서 — 반드시 이 순서로)

> ⚠️ **Phase 1이 완성 기준을 통과하기 전에 Phase 2 코드를 작성하지 말 것.** 단, 데이터 모델(3장)은 전체 Phase를 감당하도록 처음부터 설계한다.

### Phase 1 — 척추 (지금 만드는 것)
1. 그룹 라이브 세션 (코드 합류, 라운드제, 동시 공개, 세션 종료, 발행)
2. **누가 썼게 (블라인드 공개)**: 답변을 익명(가/나/다)으로 먼저 공개 → 각자 작성자 추측 → 전원 추측 완료 시 바이라인 도장 공개 + 정답 수 표시 → 발행 완료 화면에서 누적 1위 "올해의 독심술사" 발표. **3인 이상에서만 동작, 2인 세션은 자동으로 일반 공개**
3. 혼자 쓰기 (솔로 발행 — 종류 칩 + 제목, 복간과 같은 에디터)
4. 복간 (종이 회고 옮기기 — 사진 없이 텍스트만)
5. 책장 + 지난 호 열람
6. 다시 발견 — 같은 질문 다른 해 (정규화 매칭)

**완성 기준**: 폰 3대로 코드 합류 → 2라운드(추측 단계 포함) → 마감·독심술사 발표 → 책장 저장 → 다시 발견 타임라인까지 막힘 없이 동작.

### Phase 2 — 진행자와 재발견 심화
1. AI 질문 추천 (질문 팩 + 난이도 → 질문 입력 화면에서 "질문이 생각 안 나요" 버튼)
2. AI 생각 확장 (답변 작성 중 1.2초 디바운스 후 후속 질문 1개 제안 — 강요하지 않는 톤)
3. **복간 확장 — 사진 스캔 보관**: 카메라/갤러리 업로드 → 원본 이미지 그대로 "스캔 호"로 발행·열람 (이미지 저장은 R2)
4. **AI 전사**: 스캔 호 상세의 "이 호를 글로 풀기" → 비전 모델이 질문/답 초안 생성 → 사용자 확인·수정 → 구조화된 rounds로 저장(원본 사진은 pages에 유지)
5. 다시 발견 확장: 1년 전 오늘 / 오늘의 재발견(랜덤) / 올해 vs 작년 / AI 유사 질문 매칭
6. 읽기 전용 공유 링크

### Phase 3 — 잡지의 완성
1. 회고 포맷 시스템 (텍스트 기반 포맷부터: 올해의 키워드, 세 장면, 미래의 나에게 편지 → 이후 시각 포맷: 건물, 지도, 포스터)
2. 자동 편집 (세션 → 잡지 지면 자동 레이아웃) + 잡지 완성 애니메이션
3. 답변별 사진 첨부 (세션 중·복간 시)
4. **구글 폼/CSV 가져오기** (컬럼·행 매핑 UI)
5. PDF 내보내기 / 인쇄
6. AI 변화 발견 리포트

## 3. 데이터 모델 (전 Phase 공용 — 처음부터 이 구조로)

```ts
type Phase = 'lobby' | 'question' | 'answer' | 'guess' | 'ended';
type Kind = 'yearend' | 'travel' | 'monthly' | 'project' | 'reading' | 'couple' | 'free';

interface SessionMeta {
  code: string;            // 4자리, [A-HJ-NP-Z2-9]
  kind: Kind;              // 발행 시 칩 선택, 기본 yearend
  date: string;            // ISO 'YYYY-MM-DD' — 회고 기준일. 별도 year 필드 금지(표시용 연도는 파생)
  host: string;
  phase: Phase;
  roundIdx: number;        // -1 = lobby
  asker: string | null;
  question: string | null;
  history: Round[];
}

interface Round {
  asker: string;           // 질문자 이름. AI 추천 질문이어도 채택자가 asker
  question: string;
  format?: string;         // Phase 3: 포맷 ID (예: 'three-scenes'). 없으면 일반 문답
  answers: Record<string, Answer>;
}

interface Answer {
  text: string;
  media?: string[];        // Phase 3: 사진 URL
  followUps?: { q: string; a: string }[]; // Phase 2: AI 생각 확장 문답
}

interface Issue {
  id: string;
  kind: Kind;
  date: string;            // ISO
  title: string;           // 자유. 기본값 kind 기반: "2026 연말호", "제주 여행 특집호", "6월호"
  participants: string[];
  rounds: Round[];         // 스캔 호는 전사 전까지 빈 배열일 수 있음
  pages?: string[];        // Phase 2: 스캔 원본 이미지 URL (전사 후에도 원본 유지)
  source?: 'live' | 'solo' | 'paper' | 'scan' | 'import'; // 어떻게 만들어진 호인지
  shareId?: string;        // Phase 2: 읽기 전용 공유
}
```

- Phase 1에서는 `format`, `media`, `followUps`, `pages`, `shareId`를 **쓰지 않지만 스키마에 존재**시킨다 (옵셔널이므로 비용 0). `source`는 Phase 1부터 기록('live'/'solo'/'paper')
- **다시 발견은 `rounds`가 있는 호만 대상** — 스캔 호는 AI 전사를 거쳐야 타임라인에 합류
- `Answer`를 문자열이 아닌 객체로 두는 것이 Phase 2·3의 핵심 대비. Phase 1 코드에서도 `answers[name].text`로 접근할 것

## 4. 기술 스택

- **프론트: Vue 3 + TypeScript + Vite** (기존 레포 minmaxhap/recoverse 스택 유지) / **배포: Cloudflare Pages**
- **백엔드: Cloudflare Workers + KV(또는 D1)** — 세션 상태 최소 API. 실시간은 **2.5초 폴링** (Phase 1~2에서 WebSocket 금지)
- **AI(Phase 2+)**: Workers에서 Anthropic API 호출. 프롬프트에 세션 맥락(kind, 이전 질문, 작성 중 답) 전달
- 인증 없음. 세션은 4자리 코드, 개인 책장은 localStorage (공유 링크 도입 시 서버 저장 병행)
- 서버 키 구조: `session:{code}:meta`(턴 기반 단일 작성자) / 참여자·답변은 **개별 키** (동시 쓰기 안전)

## 5. 세션 상태 머신과 규칙 (Phase 1)

```
lobby → question → answer → guess(3인 이상) → (전원 추측 = 공개) → question(다음 라운드) …
                          └→ (2인: 전원 제출 = 즉시 공개)                └→ ended
```

- 질문자 로테이션: `asker = players[roundIdx % players.length]` (합류 순)
- 질문은 자기 라운드에 그 자리에서 입력. 이전 질문 목록 상시 노출 → 중복 방지
- 답변은 질문자 포함 전원, 각자 폰에서. 전원 제출 전 남의 답 노출 금지. 제출 현황 "3/4명" 표시
- **guess 단계 (3인 이상)**: 답변을 익명 라벨(가/나/다…)로 표시. 셔플 순서는 전 기기 동일해야 함 — `hash(code + roundIdx + 답변텍스트)` 기반 결정적 정렬. 자기 답은 "내 답"으로 잠금, 나머지에 작성자 배정(한 이름당 한 번). 추측은 개별 키로 저장. 전원 제출 시 공개: 바이라인이 순차적으로 도장 찍히는 연출 + "이번 라운드 2/3 적중"
- **점수**: 맞춘 수 누적. 발행 완료 화면에서 1위를 "올해의 독심술사 🏅"로 발표 (동점은 공동)
- 호스트 권한: 시작 / 다음 라운드 / 마감. **"세션 종료하기"는 모든 라이브 화면에서 호스트에게 상시 노출**. 추측 단계에서 오래 지체되면 호스트가 "그냥 공개하기" 가능
- 마감 시 각자 "내 책장에 이번 호 꽂기"로 저장 (점수는 재미 요소일 뿐, Issue에는 저장하지 않음)

## 6. 화면 목록 (Phase 1 = 01~11, 와이어프레임 번호와 일치)

1. **표지(홈)**: masthead + 커버라인 + 입구 5개(새 호 발행 / 코드로 참여 / 혼자 쓰기 / 종이 회고 옮기기 / 다시 발견) + 지난 호 목록(date 내림차순)
2. **발행/참여**: 이름, (참여 시) 코드. 발행 화면에 **종류 칩 한 줄**(연말/여행/월간/프로젝트/자유, 기본 연말). 에러는 원인+해결 안내
3. **대기실**: 초대 코드 크게, 합류자 목록(색 점+발행인 태그), 호스트만 시작(2명 이상)
4. **질문 화면**: 내 차례 = 입력+"헤드라인 확정" / 대기 = "OO이(가) 질문을 쓰는 중"+pulse. *(Phase 2: 여기에 "질문이 생각 안 나요" AI 추천 버튼)*
5. **답변 화면**: 헤드라인 + 답 입력 → 제출 후 현황 대기. *(Phase 2: 작성 중 AI 생각 확장 제안)*
6. **스프레드 — 추측(06a, 3인 이상)**: 익명 답변(가/나/다)이 나열되고, 각 답 아래 이름 칩으로 작성자 배정(자기 답은 "내 답" 잠금, 한 이름당 한 번). "추측 확정" → 현황 대기. **스프레드 — 공개(06b)**: 바이라인 도장이 순차적으로 찍히는 연출 + "이번 라운드 2/3 적중". 호스트: 다음/마감 버튼. 2인 세션은 06a 없이 즉시 06b
7. **발행 완료**: 閉 스탬프 + **"올해의 독심술사" 발표(추측 누적 1위, 동점 공동)** + "내 책장에 이번 호 꽂기". *(Phase 3: 잡지 완성 애니메이션으로 교체)*
8. **복간**: 연도·참여자 → 질문+답 반복 추가 → 지난 호로 꽂기. *(Phase 2: 진입 시 "직접 옮겨 적기 / 사진으로 보관하기" 선택. 스캔 호 상세에는 원본 이미지 뷰어 + "이 호를 글로 풀기" AI 전사 버튼)*
9. **지난 호 열람**: 잡지 지면 레이아웃(스프레드와 동일 문법, 애니메이션 없음)
10. **다시 발견**: 질문 그룹핑(공백/문장부호 정규화 매칭, 다년 우선) → date 오름차순 타임라인(연도 마커, 같은 해 복수 시 월 표기). *(Phase 2: 1년 전 오늘·랜덤 재발견 섹션, AI 유사 매칭)*
11. **혼자 쓰기**: 복간과 같은 에디터, 종류 칩+제목+내 이름, date는 오늘 → 발행

Phase 2·3 신규 화면: 공유 뷰(읽기 전용 지면), 포맷 선택, 자동 편집 결과 지면 — 착수 시점에 별도 와이어프레임 작성.

## 7. 디자인 — "에디토리얼 잡지"

**절대 금지**: 노션 / 일반 SaaS / 카드만 잔뜩 있는 대시보드 / 생산성 앱 느낌.

**지향**: 여백 많음, 큰 타이포그래피, 페이지 넘기는 느낌, 잡지를 읽는 경험, 한 장면을 보는 듯한 화면, 부드러운 애니메이션. 화면 구조는 Dashboard→Card→List가 아니라 **Library → Magazine → Chapter → Story → Memory**.

**비주얼 소재(Phase 3 포맷·편집에서 활용)**: 종이 질감, 편지, 압화, 왁스 실링, 빈티지 사진, 스크랩북, 핀, 종이 조각, 아카이브.

**토큰**
| 역할 | 값 |
|---|---|
| 종이(배경) | `#FAF7F0` |
| 잉크(텍스트/보더) | `#191511` |
| 다홍(주 액센트, CTA) | `#D8451F` |
| 금색(라벨 보조) | `#E7B355` |
| 헤어라인 | `#DED8CB` |
| 참여자 스탬프 팔레트 | `#D8451F #B98A2A #3D7A5C #4E6E93 #7D5578 #8C4B3A` (합류 순) |

**타이포그래피**: 디스플레이/인용 = Gowun Batang (헤드라인, 커버라인, 연도 마커, 답변 인용문) / UI·본문 = Pretendard. eyebrow 라벨 11px/800/letter-spacing .14em 대문자. 한국어 제목 `word-break: keep-all`.

**문법**: 직각 모서리, 잉크색 1px 보더, 헤어라인 구분선. 라벨 체계 `QUESTION 01 by 민희` / `ROUND 02` / `2026 ISSUE · 코드`. 답변 = 세리프 인용문 + 색 점·이름 바이라인. 입력 포커스 = 다홍 보더 + `2px 2px 0 #D8451F` 섀도.

**인터랙션**
- Phase 1: 스프레드 공개 순차 rise, 질문자 색 점 pulse, `prefers-reduced-motion` 존중
- Phase 2~3: 페이지 넘기기, 사진 펼쳐보기, 세션 종료 시 잡지가 완성되는 애니메이션, 책장에 꽂히는 애니메이션, 오래된 기억을 열어보는 연출

**반응형 — 데스크톱 레이아웃 (≥1024px)**

핵심 원칙: 화면이 넓어지는 게 아니라 **잡지가 펼쳐진다**. 데스크톱의 기준 단위는 "펼침면(spread)" — 가운데 접힌 선(헤어라인)을 사이에 둔 좌·우 2페이지.

- 브레이크포인트: ≤768px 모바일(단일 컬럼, 와이어프레임 01~11 그대로) / ≥1024px 데스크톱. 콘텐츠 최대 폭 1120px 중앙 정렬, 바깥은 종이 배경
- **읽는 화면은 펼침면**: 스프레드 공개(06)·지난 호 열람(09) = 왼쪽 페이지에 QUESTION 넘버·질문 헤드라인·질문자, 오른쪽 페이지에 답변 인용문 세로 흐름(답 4개 이상이면 오른쪽 2단). 다시 발견 타임라인(10b) = 왼쪽 여백에 연도 마커 고정, 오른쪽에 답변 흐름
- **쓰는 화면은 원고지**: 질문 입력(04)·답변 작성(05)·발행/참여(02)·복간·혼자 쓰기(08·11) = 최대 560px 중앙 단일 컬럼 유지. 넓게 펴지 말 것 — 입력은 종이 한 장의 집중감
- 표지(01): masthead 전폭, 그 아래 좌측에 커버라인 크게 + 우측에 입구 5개를 목차(Contents)처럼 세로 배치. 지난 호는 하단 전폭에서 책등(spine) 가로 나열 — 세로 텍스트 책등, 호버 시 살짝 뽑히는 연출
- 대기실(03)·추측(06a): 중앙 640px. 추측 화면은 모바일과 동일 구조(익명 답변 + 이름 칩)
- 데스크톱에서 추가되는 요소는 **레이아웃뿐** — 기능·카피·상태 머신은 모바일과 완전히 동일. 별도 데스크톱 전용 기능 금지

## 8. 참고
