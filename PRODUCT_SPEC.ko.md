# Recoverse 제품 명세

## 제품 정의

Recoverse는 개인과 그룹의 회고를 행성/은하처럼 기록하고, 시간이 지난 뒤 다시 탐험하거나 읽기 전용으로 공유할 수 있는 기억 우주 아카이브 서비스다.

Recoverse의 핵심은 기록 관리가 아니라 "과거의 나와 우리가 남긴 기억을 다시 탐험하는 경험"이다. 앱을 켜는 순간 사용자는 목록이나 대시보드가 아니라 자신의 기억 우주 안으로 들어와야 한다.

## 핵심 경험

> 기억을 특정 시기나 주제로 묶어 행성과 은하에 보관하고, 나중에 다시 열어 잊고 있던 나와 우리를 발견한다.

## 핵심 감정

- 따뜻함
- 그리움
- 자기이해
- 신비로움
- 조용한 발견감
- 고독하지만 외롭지 않은 탐험감

## 주요 사용자

- 연말 회고를 예쁘게 정리하고 싶은 개인 사용자
- 여행, 프로젝트, 관계, 커리어 같은 특정 주제를 기록하고 싶은 사용자
- 예전 답변을 다시 보며 자신을 이해하고 싶은 사용자
- 친구나 팀과 함께 회고를 남기고 싶은 그룹 사용자
- 나중에 일부 회고를 읽기 전용으로 공유하고 싶은 사용자

## 핵심 메타포

| 개념 | 제품 언어 |
| --- | --- |
| 홈 | 내 기억 우주 |
| 개인 캡슐 | 기억 행성 |
| 그룹 회고 | 은하 |
| 질문 카드 | 탐사 기록 / 별 |
| 답변 | 탐사 로그 |
| 공유 읽기 전용 | 관측 모드 |
| 공유 링크 | 관측 초대장 |

## 핵심 구조

```text
개인 행성
└─ 탐사 기록 / 별
   └─ 탐사 로그

그룹 은하
├─ 공통 탐사 기록
├─ 참여자 행성
└─ 참여자별 탐사 로그
```

현재 MVP 데이터는 `Capsule -> CapsuleCard -> Answer`를 유지하되, 제품 언어와 화면 구조는 `Planet -> Exploration Record -> Exploration Log`로 점진 전환한다.

## 핵심 화면

### 1. 홈 - 우주 뷰

역할:

- 앱 진입 즉시 내 기억 우주에 들어온 느낌을 만든다.
- 개인 행성과 그룹 은하를 탐험 가능한 오브젝트로 보여준다.
- 오늘의 발견을 통해 과거 기록을 다시 만나게 한다.
- `+` 버튼은 단순 추가가 아니라 개인 행성 또는 그룹 은하 생성 진입점이다.

홈에서 제외할 것:

- 연도 보기
- 빠른 입력
- 질문 비교
- JSON 가져오기/내보내기
- 일반적인 캡슐 목록 중심 UI
- 상단 탭 중심 구조

### 2. 캡슐 상세 - 행성 뷰

역할:

- 하나의 개인 행성을 탐험하는 화면이다.
- 탐사 기록/별 목록과 탐사 로그 작성 흐름을 제공한다.
- 시간여행 비교, 관측 초대장 만들기 같은 확장 기능의 위치가 된다.

### 3. 은하 - 그룹 회고

역할:

- 여러 사람이 함께 남긴 회고를 은하로 표현한다.
- 참여자는 위성 행성처럼 표현한다.
- 공통 질문과 참여자별 답변을 함께 탐험한다.

MVP 1차에서는 실제 서버/로그인 없이 IA와 데이터 모델 초안, 화면 진입 위치만 준비한다.

### 4. 관측 모드 - 공유 읽기 전용

역할:

- 공유받은 사용자가 편집 없이 행성 또는 은하를 읽는 화면이다.
- 댓글, 좋아요, SNS 피드 기능은 제공하지 않는다.
- 관측 모드는 원본 데이터가 아니라 공유 시점의 읽기 전용 스냅샷을 보여주는 방향으로 설계한다.

### 5. 아카이브/설정

역할:

- 홈에서 빠지는 보조 기능을 수용한다.
- 연도 보기, 빠른 입력, 질문 비교, JSON 관리, 언어 설정, 데이터 초기화를 이 화면으로 이동한다.

## MVP 범위

포함:

- localStorage 기반 개인 행성 기록
- 홈 우주 뷰 중심 IA
- 개인 행성 상세
- 오늘의 발견
- 탐사 기록/탐사 로그 작성
- JSON 가져오기/내보내기는 아카이브/설정에 유지
- 한국어/영어 언어 선택

제외:

- 로그인
- 클라우드 저장
- 실시간 그룹 협업
- 실제 공유 링크 발행
- PDF 내보내기
- 댓글/좋아요/SNS 피드
- Three.js 기반 3D

## 1차 리팩토링 목표

1차 리팩토링은 기능을 크게 늘리지 않고 UX 구조를 바로잡는 작업이다.

- 홈에서 목록/관리 요소를 걷어낼 준비를 한다.
- 화면 이름을 새 IA 기준으로 정리한다.
- 기존 기능은 삭제하지 않고 아카이브/설정으로 이동할 위치만 만든다.
- 개인 행성, 그룹 은하, 관측 모드를 나중에 붙일 수 있는 화면 경계를 만든다.

## 성공 기준

- 홈이 목록이나 관리 도구가 아니라 우주 공간으로 느껴진다.
- 개인 캡슐은 행성으로, 그룹 회고는 은하로 확장 가능한 구조가 된다.
- 보조 기능이 홈의 탐험 경험을 방해하지 않는다.
- 공유 확장을 위한 관측 모드 개념이 IA에 명확히 들어간다.
- 기존 localStorage 데이터와 import/export 기능은 유지된다.
