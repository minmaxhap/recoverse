# Recoverse v2 Product Plan

## 1. 제품 정의

Recoverse는 사용자가 짧은 질문에 답하면서 자신의 기억을 발견하고, 나중에 다시 보며 과거의 감정과 생각을 재발견하는 개인 회고 서비스다.

핵심 감정은 "내 기억들이 하나씩 쌓이고, 그 기억들이 모여 내가 된다"이다.

## 2. 제품 원칙

- 기본 작성은 짧고 쉽게 만든다.
- 길게 쓰기는 가능하지만 기본값은 짧은 문장이다.
- 사용자는 회고 깊이를 직접 고를 수 있다.
- 같은 질문에 대한 과거 답변을 비교할 수 있어야 한다.
- 친구 공유는 보조 기능이며, 먼저 혼자 쓰는 경험을 완성한다.
- 비주얼 테마는 바꿀 수 있지만 문구는 중립적으로 유지한다.
- 홈은 기능 목록이 아니라 기억이 쌓이는 인터랙티브 장면이어야 한다.

## 3. 주요 메뉴 구조

### 상단

- Recoverse 로고 또는 브랜드 텍스트
- 우측 프로필 영역
  - 설정
  - 언어 변경
  - 테마 변경
  - 가져오기
  - 백업
  - 로그아웃

### 하단 탭

- 기억 작성
- 홈
- 다시 보기

기본 진입 화면은 홈이다.

## 4. 테마 정책

- 기본 테마는 우주다.
- 추가 테마 후보는 방, 지도다.
- 테마 변경은 설정에서만 한다.
- 문구는 테마에 종속되지 않도록 중립적으로 쓴다.

예시 문구:

- 내 기억 공간
- 기억 묶음
- 기억 조각
- 기억 작성
- 다시 보기

## 5. 기억 작성 플로우

### 5.1 시작

첫 사용자는 홈에서 Recoverse가 "내 기억 공간"이라는 설명을 먼저 본다.

첫 홈은 기능 목록이 아니라 짧은 제품 설명과 첫 작성 CTA만 보여준다.

```text
내 기억이 쌓이는 공간

짧은 질문에 답하면
오늘의 생각과 감정이 하나의 기억으로 남아요.

나중에 다시 열어보면
그때의 나를 조금 다르게 발견할 수 있어요.

[첫 기억 작성하기]
```

친구와 함께 쓰기는 MVP 이후로 미룬다.

### 5.2 주제 선택

사용자 화면에서는 복잡한 기간 설정을 먼저 보여주지 않는다.

기본 선택지:

- 2026년
- 2026 상반기
- 2026 하반기
- 20대
- 여행
- 직접 입력

연도는 현재 연도를 기본값으로 붙인다. 다른 기간 수정은 필요할 때만 열어준다.

첫 작성의 대표 케이스는 `여행`이다.

여행을 선택하면 날짜보다 기억 이름을 먼저 묻는다.

```text
어떤 여행이었나요?

[제주 여행]
```

### 5.3 제목 생성

주제를 고르면 자동으로 기억 묶음 이름을 만든다.

예:

- 2026년의 나
- 2026 상반기의 나
- 제주 여행의 기억

이름은 나중에 수정할 수 있다.

### 5.4 깊이 선택

사용자는 작성 깊이를 고른다.

- 가볍게: 질문 5개
- 천천히: 질문 8개
- 깊게: 질문 12개
- 직접 구성: 질문 직접 선택 또는 작성

단, 첫 작성에서 `여행`을 선택한 경우에는 기본 질문 7개로 바로 시작한다.

여행 기본 질문:

1. 그 여행에서 가장 먼저 떠오르는 장면은?
2. 그때 가장 많이 느낀 감정은?
3. 같이 있던 사람이나 떠오르는 얼굴은?
4. 계속 생각나는 음식, 노래, 냄새가 있다면?
5. 예상과 달랐던 순간은?
6. 그 여행이 끝난 뒤 남은 마음은?
7. 지금 다시 돌아본다면 어떤 의미였나요?

### 5.5 질문 작성

모든 질문의 기본 입력은 짧은 문장이다.

질문은 한 화면에 하나씩 보여준다. 마지막에는 전체 답변을 한 번에 검토한다.

기본 화면:

```text
1 / 7

그 여행에서 가장 먼저 떠오르는 장면은?

[짧게 적어도 괜찮아요]

+ 더 길게 쓰기
```

태그, 선택지, 단어 입력은 보조로만 쓴다.

### 5.6 검토

저장 전에는 제목, 전체 질문/답변, 대표 장면, 대표 감정을 보여준다.

```text
제주 여행

이 기억에서 가장 또렷한 장면은?
[바다 앞에서 먹은 라면]

이 기억에 가까운 감정은?
[설렘] [편안함] [아쉬움] [자유로움] [그리움] [기타]

내 답변
1. 그 여행에서 가장 먼저 떠오르는 장면은?
답변...

[수정하기]
[저장하기]
```

대표 장면과 대표 감정은 홈 오브젝트 미리보기와 다시 보기 상세의 상단 요약에 사용한다.

### 5.7 완료

완료 화면은 다음을 포함한다.

- "기억이 완성됐어요" 계열 문구
- 내 기억 공간에 저장되는 연출
- Google로 저장하기
- Kakao로 저장하기
- 임시로 저장하기
- 임시 저장 데이터 유실 경고

저장 후에는 홈으로 이동해 새 기억 오브젝트가 생긴 것을 보여준다.

## 6. 홈 플로우

홈은 인터랙티브 장면 중심이다.

첫 진입 또는 빈 상태에서는 짧은 설명과 첫 작성 CTA만 보여준다.

홈에서 보여줄 것:

- 내가 만든 기억 묶음
- 시간순 구조
- 장면 안 오브젝트 형태의 표현
- 선택한 기억의 미리보기

예:

```text
2026
- 2026년의 나
- 2026 상반기의 나
- 2026 하반기의 나
```

각 기억 묶음은 장면 속 오브젝트처럼 배치한다.

MVP 오브젝트는 단순한 원/별 형태로 시작한다. 이후 여행/연도/20대별 감성 오브젝트로 확장하고, 마지막 단계에서 드래그/회전/탐험형 인터랙티브 공간으로 확장한다.

홈 미리보기는 다음 정보만 보여준다.

```text
제주 여행

자유로움

"바다 앞에서 먹은 라면"

[열어보기]
```

홈에서는 공유 CTA를 노출하지 않는다.

## 7. 다시 보기 플로우

다시 보기의 기본은 회고 목록에서 고르는 방식이다.

추가로 같은 질문의 답변 비교를 제공한다.

예:

```text
2026
[올해 회고]
[상반기 회고]
[하반기 회고]

같은 질문 비교
"가장 기억나는 순간은?"
상반기 답변 / 하반기 답변 / 올해 답변
```

사용자는 같은 해의 올해, 상반기, 하반기 회고를 함께 볼 수 있어야 한다.

상세 화면은 대표 장면과 대표 감정을 먼저 크게 보여주고, 아래에 질문/답변 전체를 배치한다.

```text
제주 여행

자유로움

"바다 앞에서 먹은 라면"

그 여행에서 가장 먼저 떠오르는 장면은?
답변...

그때 가장 많이 느낀 감정은?
답변...

[공유하기]
```

## 8. 공유 플로우

MVP 공유는 읽기 전용 링크만 포함한다.

공유는 다시 보기 상세 화면에서 제공한다.

친구가 공유 링크로 들어오면 다음을 할 수 있다.

- 내 회고 읽기

공유 MVP 흐름:

```text
내가 쓴 기억 상세
↓
[공유하기]
↓
읽기 전용 링크 생성
↓
친구가 링크 진입
↓
공유된 회고 읽기
↓
[나도 기억 작성하기]
```

공유 화면 문구:

```text
OO님의 기억을 보고 있어요

이 화면에서는 읽기만 할 수 있어요.
```

MVP 이후 확장:

```text
친구도 같은 질문에 답하기
↓
서로 답한 뒤 비교 공개
```

친구 답변 확장 흐름:

```text
친구가 링크 진입
↓
공유된 회고 읽기
↓
[나도 같은 질문에 답하기]
↓
이름 입력
↓
질문별 짧은 답변
↓
서로 답한 뒤 비교 공개
```

## 9. 로그인과 백업 정책

작성 시작은 로그인 없이 가능하다.

로그인이 필요한 순간:

- 안전 저장
- 백업
- 기기 동기화

첫 작성은 로그인 없이 가능하지만, 저장 시점에는 로그인을 유도한다.

지원 로그인:

- Google
- Kakao

로컬 저장은 임시 저장 용도로만 본다. 캐시 삭제 등으로 데이터가 사라질 수 있으므로 임시 저장에는 명확한 경고를 표시한다.

완료 화면 CTA:

- Google로 저장하기
- Kakao로 저장하기
- 임시로 저장하기

임시 저장 안내:

```text
브라우저 데이터가 삭제되면 임시 저장한 기억은 사라질 수 있어요.
```

백업은 두 방식을 모두 지원한다.

- JSON 내보내기/가져오기
- 계정 기반 서버 저장

## 10. MVP 범위

### 포함

- 상단 브랜드/프로필 메뉴
- 하단 탭: 기억 작성 / 홈 / 다시 보기
- 기본 진입: 홈
- 홈 인터랙티브 장면
- 홈 오브젝트 미리보기
- 기억 작성 wizard
- 주제 선택
- 첫 작성 여행 회고 7문항
- 짧은 문장 답변
- 마지막 전체 검토
- 대표 장면/감정 선택
- 자동 제목 생성
- 완료 연출
- 다시 보기 목록
- 다시 보기 상세 상단 대표 장면/감정
- 같은 해 회고 묶음
- 같은 질문 답변 비교
- 읽기 전용 공유 링크
- Google/Kakao 로그인 저장 유도
- JSON 백업/가져오기

### 나중에

- 작성 깊이 선택 고도화
- 친구의 같은 질문 답변
- 테마별 고급 인터랙션
- 방/지도 테마 완성도 강화
- 친구 질문 추가
- 댓글/반응
- 답변 승인 모드
- 질문 세트 마켓/추천
- 서버 동기화 고도화
