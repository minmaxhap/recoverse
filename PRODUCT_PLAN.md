# Recoverse v2 Product Plan

## 1. 제품 정의

Recoverse는 사용자가 짧은 질문에 답하면서 자신의 기억을 발견하고, 나중에 다시 보며 과거의 감정과 생각을 재발견하는 개인 회고 서비스다.

핵심 감정은 "내 기억들이 하나씩 쌓이고, 그 기억들이 모여 내가 된다"이다.

## 2. 제품 원칙

- 기본 작성은 짧고 쉽게 만든다.
- 길게 쓰기는 가능하지만 기본값은 짧은 문장이다.
- 사용자는 회고 깊이를 직접 고를 수 있다.
- 같은 질문에 대한 과거 답변을 비교할 수 있어야 한다.
- 친구 공유는 보조 기능이며, 먼저 혼자 쓰는 경험을 완성한다.
- 비주얼 테마는 바꿀 수 있지만 문구는 중립적으로 유지한다.
- 홈은 기능 목록이 아니라 기억이 쌓이는 인터랙티브 장면이어야 한다.

## 3. 주요 메뉴 구조

### 상단

- Recoverse 로고 또는 브랜드 텍스트
- 우측 프로필 영역
  - 설정
  - 언어 변경
  - 테마 변경
  - 가져오기
  - 백업
  - 로그아웃

### 하단 탭

- 기억 작성
- 홈
- 다시 보기

기본 진입 화면은 홈이다.

## 4. 테마 정책

- 기본 테마는 우주다.
- 추가 테마 후보는 방, 지도다.
- 테마 변경은 설정에서만 한다.
- 문구는 테마에 종속되지 않도록 중립적으로 쓴다.

예시 문구:

- 내 기억 공간
- 기억 묶음
- 기억 조각
- 기억 작성
- 다시 보기

## 5. 기억 작성 플로우

### 5.1 시작

첫 사용자는 혼자 기억 작성부터 시작한다.

친구와 함께 쓰기는 완료 이후 자연스럽게 제안한다.

### 5.2 주제 선택

사용자 화면에서는 복잡한 기간 설정을 먼저 보여주지 않는다.

기본 선택지:

- 2026년
- 2026 상반기
- 2026 하반기
- 20대
- 여행
- 직접 입력

연도는 현재 연도를 기본값으로 붙인다. 다른 기간 수정은 필요할 때만 열어준다.

### 5.3 제목 생성

주제를 고르면 자동으로 기억 묶음 이름을 만든다.

예:

- 2026년의 나
- 2026 상반기의 나
- 제주 여행의 기억

이름은 나중에 수정할 수 있다.

### 5.4 깊이 선택

사용자는 작성 깊이를 고른다.

- 가볍게: 질문 5개
- 천천히: 질문 8개
- 깊게: 질문 12개
- 직접 구성: 질문 직접 선택 또는 작성

### 5.5 질문 작성

모든 질문의 기본 입력은 짧은 문장이다.

기본 화면:

```text
올해 가장 먼저 떠오르는 순간은?

[짧게 적어도 괜찮아요]

+ 더 길게 쓰기
```

태그, 선택지, 단어 입력은 보조로만 쓴다.

### 5.6 완료

완료 화면은 다음을 포함한다.

- "기억이 완성됐어요" 계열 문구
- 내 기억 공간에 저장되는 연출
- 크게: 로그인하고 안전하게 저장
- 작게: 나중에 하기
- 작게: 친구에게 같은 질문 보내기

## 6. 홈 플로우

홈은 인터랙티브 장면 중심이다.

홈에서 보여줄 것:

- 내가 만든 기억 묶음
- 시간순 구조
- 장면 안 오브젝트 형태의 표현

예:

```text
2026
- 2026년의 나
- 2026 상반기의 나
- 2026 하반기의 나
```

각 기억 묶음은 장면 속 오브젝트처럼 배치한다.

MVP의 첫 강한 인터랙션은 홈 장면에 적용한다.

## 7. 다시 보기 플로우

다시 보기의 기본은 회고 목록에서 고르는 방식이다.

추가로 같은 질문의 답변 비교를 제공한다.

예:

```text
2026
[올해 회고]
[상반기 회고]
[하반기 회고]

같은 질문 비교
"가장 기억나는 순간은?"
상반기 답변 / 하반기 답변 / 올해 답변
```

사용자는 같은 해의 올해, 상반기, 하반기 회고를 함께 볼 수 있어야 한다.

## 8. 공유 플로우

공유는 두 위치에서 제공한다.

- 회고 완료 직후: 작게 제안
- 다시 보기 상세 화면: 명확한 액션으로 제공

친구가 공유 링크로 들어오면 다음을 할 수 있다.

- 내 회고 읽기
- 같은 질문에 답하기

친구 답변은 서로 답해야 공개된다.

공유 MVP 흐름:

```text
친구가 링크 진입
↓
내 회고 읽기
↓
[나도 같은 질문에 답하기]
↓
이름 입력
↓
질문별 짧은 답변
↓
서로 답한 뒤 비교 공개
```

## 9. 로그인과 백업 정책

작성 시작은 로그인 없이 가능하다.

로그인이 필요한 순간:

- 공유
- 백업
- 기기 동기화

로컬 저장은 임시 저장 용도로만 본다. 캐시 삭제 등으로 데이터가 사라질 수 있으므로 첫 회고 저장 완료 후 중간 강도로 로그인 저장을 유도한다.

완료 화면 CTA:

- 크게: 로그인하고 안전하게 저장
- 작게: 나중에 하기

백업은 두 방식을 모두 지원한다.

- JSON 내보내기/가져오기
- 계정 기반 서버 저장

## 10. MVP 범위

### 포함

- 상단 브랜드/프로필 메뉴
- 하단 탭: 기억 작성 / 홈 / 다시 보기
- 기본 진입: 홈
- 홈 인터랙티브 장면
- 기억 작성 wizard
- 주제 선택
- 깊이 선택
- 짧은 문장 답변
- 자동 제목 생성
- 완료 연출
- 다시 보기 목록
- 같은 해 회고 묶음
- 같은 질문 답변 비교
- 공유 링크 읽기
- 친구의 같은 질문 답변
- 로그인 저장 유도
- JSON 백업/가져오기

### 나중에

- 테마별 고급 인터랙션
- 방/지도 테마 완성도 강화
- 친구 질문 추가
- 댓글/반응
- 답변 승인 모드
- 질문 세트 마켓/추천
- 서버 동기화 고도화
