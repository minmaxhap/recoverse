# Recoverse Product Plan

## 1. 제품 정의

Recoverse는 사용자가 짧은 질문에 답하면서 오늘의 감정과 기억을 보관하고, 시간이 지난 뒤 다시 열어보며 과거의 나를 발견하는 모바일 회고 서비스다.

새 제품 방향은 **Story Book x Time Capsule**이다. 회고는 로그가 아니라 편지이고, 목록은 피드가 아니라 책장이다.

## 2. 제품 원칙

- 기본 작성은 30초 안에 끝난다.
- 회고는 짧게 시작하고, 필요할 때만 깊어진다.
- 저장/완료는 봉인되는 경험처럼 느껴져야 한다.
- 홈은 기능 목록이 아니라 오늘 쓸 이유와 과거 회고의 책장을 보여준다.
- 연말, 여행, 친구 비교는 별도 기능처럼 튀지 않고 같은 책/편지 언어 안에서 확장된다.
- 공유는 보조 기능이며, 먼저 혼자 쓰고 다시 보는 경험을 완성한다.
- 기존 `Reflection` 데이터와 `recoverse_reflections_v1` 저장 계약은 유지한다.

## 4. 네비게이션

기본 하단 탭은 새 디자인에서 다음 구조를 우선한다.

```text
나의 책고 / 작성하기 / 둘러보기
```

기능 구현 단계에서는 기존 `홈 / 기억 작성 / 다시 보기` 라벨을 임시 유지할 수 있지만, 최종 문구는 제품 톤에 맞게 위 구조로 정리한다.

상단은 브랜드와 설정 진입만 둔다. 기능 설명형 메뉴는 줄인다.

## 5. 작성 플로우

### 5.1 Home에서 시작

사용자는 Home에서 바로 30초 회고를 시작할 수 있어야 한다.

```text
30초면 충분해요.

질문에 답하다 보면 오늘 하루도, 여행도, 연말도 하나의 추억이 됩니다.

[30초 만에 시작하기]
[예전 회고 보기]
```

### 5.2 회고 시작

사용자는 상황에 맞는 회고 모드를 고른다.

- 30초 회고: 기본값. 오늘을 짧게 닫기.
- 연말 회고: 한 해를 묶기.
- 여행 회고: 장소와 감정을 함께 저장.
- 친구 비교: 같은 질문을 서로 다른 답변으로 보기.

### 5.3 질문 작성

질문은 한 화면에 하나씩 보여준다. 기본 입력은 짧은 문장이다.

```text
오늘의 질문 1 / 3

오늘 나를 가장 오래 붙잡은 생각은 무엇이었나요?

[편지지 입력 영역]
[회고 봉인하기]
```

### 5.4 완료

완료 화면은 `저장됨`보다 `봉인됨`을 강조한다.

```text
회고가 봉인됐어요.

오늘의 문장은 나중에 다시 열 수 있어요.

[앨범에 보관하기]
[하나 더 쓰기]
```

## 6. 다시 보기와 앨범

`다시 보기`는 새 디자인에서 `회고 앨범`으로 재해석한다.

앨범 묶음:

- 월별 회고
- 여행 회고
- 연말 회고
- 친구와 같은 질문

상세 화면은 편지 한 장처럼 읽혀야 한다. 상단에는 날짜/감정/대표 질문, 본문에는 회고 문장, 하단에는 태그와 사진 조각을 둔다.

## 7. 공유와 친구 비교

MVP 공유는 기존처럼 URL 해시 기반 읽기 전용 스냅샷을 유지한다.

친구 비교는 장기적으로 다음 흐름을 목표로 한다.

```text
같은 질문 선택
내 답변 작성
친구 초대
서로 답한 뒤 비교 공개
```

초기 구현에서는 친구 데이터 없이 샘플 비교 화면 또는 로컬 비교만 제공할 수 있다.

## 8. 설정과 저장

설정 화면은 기능 목록이어도 시각 언어를 유지한다.

- 저녁 질문 알림
- 잠금 보관함
- 종이 테마
- 내보내기/가져오기
- 계정 저장 준비

계정 저장 정책은 [ACCOUNT_STORAGE_PLAN.ko.md](./ACCOUNT_STORAGE_PLAN.ko.md)를 따른다.

## 9. MVP 전환 범위

### 이번 디자인 전환에 포함

- 새 디자인 토큰과 CSS 변수
- Home의 Story Book x Time Capsule 리디자인
- 30초 회고 진입
- 질문 작성 편지지 UI
- 작성 완료 seal UI
- 회고 앨범과 회고 상세
- 연말/여행/친구 비교 화면의 최소 구현 또는 화면 골격
- 설정 화면의 새 디자인 적용

- PDF 내보내기
- 이미지 업로드

- 3D/WebGL 화면
