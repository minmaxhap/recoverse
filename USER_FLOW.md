# Recoverse v2 User Flow

## 1. 전체 구조

Recoverse의 기본 이동 구조는 세 개의 하단 탭과 하나의 상단 프로필 메뉴로 구성한다.

```text
상단
Recoverse 브랜드 / 프로필 메뉴

하단
기억 작성 / 홈 / 다시 보기
```

기본 진입은 홈이다.

## 2. 최상위 IA

```text
Recoverse
├─ 홈
│  ├─ 기억 공간 장면
│  ├─ 연도별 기억 묶음
│  └─ 기억 묶음 열기
├─ 기억 작성
│  ├─ 주제 선택
│  ├─ 깊이 선택
│  ├─ 질문 답변
│  └─ 완료/저장
├─ 다시 보기
│  ├─ 기억 묶음 목록
│  ├─ 상세 보기
│  ├─ 같은 질문 비교
│  └─ 공유
└─ 프로필 메뉴
   ├─ 설정
   ├─ 언어 변경
   ├─ 테마 변경
   ├─ 가져오기
   ├─ 백업
   └─ 로그아웃
```

## 3. 첫 진입 플로우

목표: 사용자가 Recoverse가 "기억이 쌓이는 공간"이라는 것을 바로 이해한다.

```text
앱 진입
↓
홈 표시
↓
내 기억 공간 장면 표시
↓
기억 묶음이 있으면 연도별 오브젝트 표시
↓
기억 묶음이 없으면 첫 기억 작성 유도
```

### 빈 상태

```text
아직 남긴 기억이 없어요.
짧게 하나만 남겨볼까요?

[기억 작성하기]
```

### 기존 데이터가 있는 상태

```text
내 기억 공간

2026
- 2026년의 나
- 2026 상반기의 나
- 2026 하반기의 나
```

## 4. 홈 플로우

목표: 사용자가 자신이 남긴 기억 묶음을 시간순으로 보고, 장면 속 오브젝트처럼 탐색한다.

```text
홈
↓
연도별 기억 묶음 확인
↓
기억 묶음 오브젝트 선택
↓
기억 묶음 미리보기
↓
[열어보기] 선택
↓
다시 보기 상세로 이동
```

### 홈에서 가능한 행동

- 기억 묶음 열기
- 하단 탭으로 기억 작성 이동
- 하단 탭으로 다시 보기 이동
- 프로필 메뉴 열기

### 홈에서 하지 않는 것

- 긴 설명 노출
- 복잡한 통계 노출
- 공유 CTA 전면 노출
- 설정 기능 전면 노출

## 5. 기억 작성 플로우

목표: 사용자가 부담 없이 짧은 문장으로 기억을 남긴다.

```text
기억 작성 탭
↓
무엇을 돌아볼지 선택
↓
자동 이름 생성
↓
얼마나 깊게 볼지 선택
↓
질문에 하나씩 답변
↓
완료 화면
↓
안전 저장 유도
↓
홈에 기억 묶음 추가
```

### 5.1 주제 선택

```text
무엇을 돌아볼까요?

[2026년]
[2026 상반기]
[2026 하반기]
[20대]
[여행]
[직접 입력]
```

기본값은 현재 연도를 기준으로 만든다.

예:

- 2026년
- 2026 상반기
- 2026 하반기

직접 입력은 날짜가 아니어도 된다.

예:

- 스물아홉
- 제주 여행
- 퇴사 전후

### 5.2 자동 이름 생성

```text
선택: 2026 상반기
생성: 2026 상반기의 나
```

자동 이름은 작성 흐름을 끊지 않는다. 수정은 상세 또는 완료 이후에 한다.

### 5.3 깊이 선택

```text
얼마나 깊게 볼까요?

[가볍게] 질문 5개
[천천히] 질문 8개
[깊게] 질문 12개
[직접 구성] 질문 직접 선택/작성
```

### 5.4 질문 답변

한 화면에는 질문 하나만 보여준다.

```text
3 / 8

가장 먼저 떠오르는 순간은?

[짧게 적어도 괜찮아요]

+ 더 길게 쓰기

[이전] [다음]
```

기본 답변은 짧은 문장이다. 길게 쓰기는 선택 옵션이다.

### 5.5 작성 중 이동

작성 중 하단 탭을 누르면 확인한다.

```text
작성 중인 기억이 있어요.
이동하면 이어서 작성할 수 있어요.

[계속 작성]
[이동하기]
```

### 5.6 완료

```text
기억이 완성됐어요.

내 기억 공간에 저장할게요.

[로그인하고 안전하게 저장]
[나중에 하기]

친구들은 이 질문에 어떻게 답할까요?
[친구에게 같은 질문 보내기]
```

로그인 유도는 중간 강도다. 작성 완료 자체를 막지 않는다.

## 6. 다시 보기 플로우

목표: 사용자가 작성한 회고를 쉽게 다시 열고, 같은 질문에 대한 과거 답변을 비교한다.

```text
다시 보기 탭
↓
연도별 기억 묶음 목록
↓
기억 묶음 선택
↓
상세 보기
↓
질문/답변 읽기
↓
같은 질문 비교 또는 공유
```

### 6.1 목록

```text
2026
[2026년의 나]
[2026 상반기의 나]
[2026 하반기의 나]

여행
[제주 여행의 기억]
```

같은 해의 올해, 상반기, 하반기는 함께 보이게 한다.

### 6.2 상세

```text
2026 상반기의 나

1. 가장 먼저 떠오르는 순간은?
답변...

2. 그때 가장 많이 느낀 감정은?
답변...

[같은 질문 비교]
[공유하기]
```

### 6.3 같은 질문 비교

```text
질문
가장 먼저 떠오르는 순간은?

2026 상반기
답변...

2026 하반기
답변...

2026년
답변...
```

비교는 "내가 이 시기에 어떻게 달라졌는지"를 발견하게 만드는 기능이다.

## 7. 공유 플로우

목표: 친구가 내 회고를 읽고, 같은 질문에 답해 서로의 기억을 비교한다.

공유 진입점:

- 완료 화면의 작은 제안
- 다시 보기 상세의 명확한 공유 버튼

```text
공유하기
↓
로그인 여부 확인
↓
로그인 필요 시 로그인 유도
↓
공유 링크 생성
↓
친구에게 전달
```

### 7.1 친구 링크 진입

```text
친구가 링크 진입
↓
공유된 회고 읽기
↓
[나도 같은 질문에 답하기]
↓
이름 입력
↓
질문별 짧은 답변
↓
제출
```

### 7.2 답변 공개 규칙

친구 답변은 서로 답해야 공개된다.

```text
내 답변 있음 + 친구 답변 있음
↓
서로 답변 비교 공개
```

친구가 아직 답하지 않은 상태에서는 내 답변만 보여주거나, 비교 영역을 잠금 상태로 둔다.

## 8. 로그인/백업 플로우

### 8.1 로그인 필요 시점

로그인은 처음부터 요구하지 않는다.

필요한 순간:

- 공유
- 백업
- 기기 동기화

### 8.2 첫 저장 이후

```text
첫 기억 작성 완료
↓
로컬 임시 저장
↓
안전 저장 안내
↓
[로그인하고 안전하게 저장]
[나중에 하기]
```

### 8.3 백업

```text
프로필 메뉴
↓
백업
↓
JSON 내보내기 또는 계정 저장
```

### 8.4 가져오기

```text
프로필 메뉴
↓
가져오기
↓
JSON 파일 선택
↓
미리보기
↓
가져오기 완료
```

## 9. 프로필 메뉴 플로우

```text
상단 프로필 영역 선택
↓
메뉴 열림
↓
설정 / 언어 변경 / 테마 변경 / 가져오기 / 백업 / 로그아웃 선택
```

### 테마 변경

```text
프로필 메뉴
↓
테마 변경
↓
우주 / 방 / 지도 선택
↓
홈 장면 테마 변경
```

문구는 테마와 무관하게 중립적으로 유지한다.

## 10. MVP 구현 순서

1. 상단/하단 내비게이션 구조 정리
2. 홈 기본 진입과 기억 묶음 장면 설계
3. 기억 작성 wizard 구현
4. 완료 화면과 로그인 저장 유도 구현
5. 다시 보기 목록/상세 구현
6. 같은 질문 비교 구현
7. JSON 백업/가져오기 정리
8. 공유 링크 읽기/같은 질문 답변 구현
9. 프로필 메뉴와 테마 변경 구현
