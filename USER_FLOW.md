# Recoverse User Flow

## 1. 전체 구조

Recoverse의 새 흐름은 **나의 책고 / 작성하기 / 둘러보기**를 중심으로 한다. 현재 코드의 `홈 / 기억 작성 / 다시 보기` 탭은 구현 전환 중 임시로 유지할 수 있지만, 최종 제품 언어는 책과 편지 은유에 맞춘다.

```text
Recoverse
├─ Splash
├─ Home
│  ├─ 30초 회고 CTA
│  ├─ 샘플 회고 카드
│  ├─ 회고 앨범 진입
│  └─ 친구 비교 진입
├─ 작성하기
│  ├─ 회고 시작
│  ├─ 질문 작성
│  └─ 작성 완료
├─ 나의 책고
│  ├─ 회고 앨범
│  ├─ 회고 상세
│  ├─ 연말 회고
│  └─ 여행 회고
├─ 둘러보기
│  └─ 친구 비교
├─ 설정
└─ 디자인 시스템
```

## 2. 첫 진입

목표: 사용자가 Recoverse를 “기억을 보관하는 책과 타임캡슐”로 이해하고 바로 첫 문장을 남기게 한다.

```text
Splash
↓
Home
  “30초면 충분해요.”
  [30초 만에 시작하기]
  [예전 회고 보기]
```

첫 진입에서 하지 않는 것:

- 긴 온보딩
- 복잡한 기능 목록
- 계정 가입 요구
- 테마 선택 강요

## 3. 30초 회고 작성

```text
Home
↓
회고 시작
  30초 회고 선택
↓
질문 작성
  질문 1개 또는 3개
  편지지 입력
  감정 칩 선택
↓
작성 완료
  왁스 실링 완료 연출
  앨범에 보관
↓
회고 상세 또는 Home
```

질문 작성 화면 원칙:

- 질문은 하나씩 보여준다.
- 입력은 편지지처럼 보이되 실제 textarea/입력 컴포넌트여야 한다.
- 자동 저장 상태와 글자 수를 작게 보여준다.
- 완료 CTA는 `회고 봉인하기` 계열 문구를 사용한다.

## 4. 회고 앨범 흐름

```text
Home 또는 하단 탭
↓
회고 앨범
  월별 회고
  여행 회고
  친구와 같은 질문
  연말 회고
↓
회고 상세
  편지형 본문
  태그
  사진 조각
  이어 쓰기
```

앨범은 목록이라기보다 책장이다. 회고 묶음은 카드/책/엽서 표지처럼 보여야 한다.

## 5. 연말 회고

```text
회고 앨범
↓
연말 회고
  올해 작성률
  많이 등장한 감정
  자주 나온 말
  월별 하이라이트
```

연말 회고의 숫자는 차갑게 보이면 안 된다. 통계는 책갈피처럼 작게, 문장은 본문처럼 크게 둔다.

## 6. 여행 회고

```text
회고 시작
↓
여행 회고 선택
↓
여행 이름 또는 장소 입력
↓
질문 작성
↓
여행 회고 상세
  엽서형 지도
  장소별 기억
  사진 조각
```

지도는 내비게이션이 아니라 엽서다. 정확한 지리보다 장소와 감정의 연결이 중요하다.

## 7. 친구 비교

```text
친구 비교
↓
같은 질문 확인
↓
내 답변과 친구 답변 비교
↓
공통 키워드 확인
```

MVP에서는 실제 친구 입력이 없으면 샘플/로컬 데이터로 화면 골격만 구성한다. 비교는 점수화가 아니라 “같은 질문을 다른 문장으로 읽는 경험”이다.

## 8. 설정

```text
설정
├─ 저녁 질문 알림
├─ 잠금 보관함
├─ 종이 테마
├─ 내보내기
├─ 가져오기
└─ 계정 저장 준비
```

설정도 새 디자인 톤을 유지한다. 기능 행은 종이 카드처럼 보이고, 위험 행동은 별도 확인 단계를 둔다.

## 9. 공유

기존 URL 해시 기반 읽기 전용 공유는 유지한다.

```text
회고 상세
↓
공유하기
↓
읽기 전용 스냅샷 생성
↓
친구가 링크 진입
↓
공유 회고 읽기
```

공유된 화면에서도 편지/책장 언어를 유지한다. 원본 수정, 삭제, 좋아요, 댓글은 제공하지 않는다.

## 10. 구현 순서

1. 디자인 토큰과 전역 CSS 정리
2. Splash/Home 리디자인
3. 회고 시작/질문 작성/작성 완료 리디자인
4. 회고 앨범/회고 상세 리디자인
5. 연말/여행/친구 비교 화면 골격 추가
6. 설정 화면 리디자인
7. 디자인 시스템 화면 추가
8. 기존 공유/백업/저장 흐름 재검증
