# Recoverse v2 User Flow

## 1. 전체 구조

Recoverse의 기본 이동 구조는 세 개의 하단 탭과 하나의 상단 프로필 메뉴로 구성한다.

```text
상단
Recoverse 브랜드 / 프로필 메뉴

하단
기억 작성 / 홈 / 다시 보기
```

기본 진입은 홈이다.

## 2. 최상위 IA

```text
Recoverse
├─ 홈
│  ├─ 기억 공간 장면
│  ├─ 연도별 기억 묶음
│  └─ 기억 묶음 열기
├─ 기억 작성
│  ├─ 주제 선택
│  ├─ 여행 이름 입력
│  ├─ 질문 답변
│  ├─ 전체 답변 검토
│  ├─ 대표 장면/감정 선택
│  └─ 완료/저장
├─ 다시 보기
│  ├─ 기억 묶음 목록
│  ├─ 상세 보기
│  ├─ 같은 질문 비교
│  └─ 공유
└─ 프로필 메뉴
   ├─ 설정
   ├─ 언어 변경
   ├─ 테마 변경
   ├─ 가져오기
   ├─ 백업
   └─ 로그아웃
```

## 3. 첫 진입 플로우

목표: 사용자가 Recoverse가 "기억이 쌓이는 공간"이라는 것을 바로 이해한다.

```text
앱 진입
↓
홈 표시
↓
내 기억 공간 설명 표시
↓
첫 기억 작성 CTA 표시
↓
기억 묶음이 있으면 설명 아래 또는 장면 안에 오브젝트 표시
↓
기억 묶음이 없으면 첫 기억 작성 유도
```

### 빈 상태

```text
내 기억이 쌓이는 공간

짧은 질문에 답하면
오늘의 생각과 감정이 하나의 기억으로 남아요.

나중에 다시 열어보면
그때의 나를 조금 다르게 발견할 수 있어요.

[첫 기억 작성하기]
```

### 기존 데이터가 있는 상태

```text
내 기억 공간

2026
- 2026년의 나
- 2026 상반기의 나
- 2026 하반기의 나
```

## 4. 홈 플로우

목표: 사용자가 자신이 남긴 기억 묶음을 시간순으로 보고, 장면 속 오브젝트처럼 탐색한다.

```text
홈
↓
연도별 기억 묶음 확인
↓
기억 묶음 오브젝트 선택
↓
기억 묶음 미리보기
↓
[열어보기] 선택
↓
다시 보기 상세로 이동
```

### 홈에서 가능한 행동

- 기억 묶음 열기
- 하단 탭으로 기억 작성 이동
- 하단 탭으로 다시 보기 이동
- 프로필 메뉴 열기

### 홈에서 하지 않는 것

- 긴 기능 설명 노출
- 복잡한 통계 노출
- 공유 CTA 전면 노출
- 설정 기능 전면 노출

## 5. 기억 작성 플로우

목표: 사용자가 부담 없이 짧은 문장으로 기억을 남긴다.

```text
기억 작성 탭
↓
무엇을 돌아볼지 선택
↓
여행 선택 시 여행 이름 입력
↓
자동 이름 생성
↓
질문에 하나씩 답변
↓
전체 답변 검토
↓
대표 장면/감정 선택
↓
저장 화면
↓
안전 저장 유도
↓
홈에 기억 묶음 추가
```

### 5.1 주제 선택

```text
무엇을 돌아볼까요?

[2026년]
[2026 상반기]
[2026 하반기]
[20대]
[여행]
[직접 입력]
```

기본값은 현재 연도를 기준으로 만든다.

예:

- 2026년
- 2026 상반기
- 2026 하반기

직접 입력은 날짜가 아니어도 된다.

예:

- 스물아홉
- 제주 여행
- 퇴사 전후

첫 작성의 대표 케이스는 여행이다.

여행을 선택하면 날짜 설정 대신 여행 이름을 먼저 묻는다.

```text
어떤 여행이었나요?

[제주 여행]
```

### 5.2 자동 이름 생성

예:

- 선택: 2026 상반기 -> 생성: 2026 상반기의 나
- 입력: 제주 여행 -> 생성: 제주 여행

자동 이름은 작성 흐름을 끊지 않는다. 수정은 상세 또는 완료 이후에 한다.

### 5.3 깊이 선택

첫 작성의 여행 회고는 깊이 선택 없이 기본 7문항으로 시작한다.

여행 기본 질문:

1. 그 여행에서 가장 먼저 떠오르는 장면은?
2. 그때 가장 많이 느낀 감정은?
3. 같이 있던 사람이나 떠오르는 얼굴은?
4. 계속 생각나는 음식, 노래, 냄새가 있다면?
5. 예상과 달랐던 순간은?
6. 그 여행이 끝난 뒤 남은 마음은?
7. 지금 다시 돌아본다면 어떤 의미였나요?

일반 회고에서는 작성 깊이를 고를 수 있다.

```text
얼마나 깊게 볼까요?

[가볍게] 질문 5개
[천천히] 질문 8개
[깊게] 질문 12개
[직접 구성] 질문 직접 선택/작성
```

### 5.4 질문 답변

한 화면에는 질문 하나만 보여준다.

```text
1 / 7

그 여행에서 가장 먼저 떠오르는 장면은?

[짧게 적어도 괜찮아요]

+ 더 길게 쓰기

[이전] [다음]
```

기본 답변은 짧은 문장이다. 길게 쓰기는 선택 옵션이다.

### 5.5 전체 검토

마지막 질문까지 답하면 전체 답변을 한 화면에서 검토한다.

```text
제주 여행

내 답변
1. 그 여행에서 가장 먼저 떠오르는 장면은?
답변...

2. 그때 가장 많이 느낀 감정은?
답변...

[수정하기]
[다음]
```

### 5.6 대표 장면/감정 선택

```text
이 기억에서 가장 또렷한 장면은?
[바다 앞에서 먹은 라면]

이 기억에 가까운 감정은?
[설렘] [편안함] [아쉬움] [자유로움] [그리움] [기타]

[저장하기]
```

대표 장면과 감정은 홈 미리보기와 다시 보기 상세의 상단 요약에 사용한다.

### 5.7 작성 중 이동

작성 중 하단 탭을 누르면 확인한다.

```text
작성 중인 기억이 있어요.
이동하면 이어서 작성할 수 있어요.

[계속 작성]
[이동하기]
```

### 5.8 저장

```text
기억이 완성됐어요.

안전하게 보관하려면 로그인이 필요해요.
로그인하면 다른 기기에서도 다시 볼 수 있어요.

[Google로 저장하기]
[Kakao로 저장하기]

[임시로 저장하기]

브라우저 데이터가 삭제되면 임시 저장한 기억은 사라질 수 있어요.
```

저장 후에는 홈으로 이동하고, 새 기억 오브젝트가 생긴 상태를 보여준다.

## 6. 다시 보기 플로우

목표: 사용자가 작성한 회고를 쉽게 다시 열고, 같은 질문에 대한 과거 답변을 비교한다.

```text
다시 보기 탭
↓
연도별 기억 묶음 목록
↓
기억 묶음 선택
↓
상세 보기
↓
질문/답변 읽기
↓
같은 질문 비교 또는 공유
```

### 6.1 목록

```text
2026
[2026년의 나]
[2026 상반기의 나]
[2026 하반기의 나]

여행
[제주 여행의 기억]
```

같은 해의 올해, 상반기, 하반기는 함께 보이게 한다.

### 6.2 상세

```text
제주 여행

자유로움

"바다 앞에서 먹은 라면"

그 여행에서 가장 먼저 떠오르는 장면은?
답변...

그때 가장 많이 느낀 감정은?
답변...

[같은 질문 비교]
[공유하기]
```

### 6.3 같은 질문 비교

```text
질문
가장 먼저 떠오르는 순간은?

2026 상반기
답변...

2026 하반기
답변...

2026년
답변...
```

비교는 "내가 이 시기에 어떻게 달라졌는지"를 발견하게 만드는 기능이다.

## 7. 공유 플로우

목표: 친구가 내 회고를 읽기 전용으로 확인하고, 원하면 자신의 기억 작성을 시작한다.

공유 진입점:

- 다시 보기 상세의 명확한 공유 버튼

```text
공유하기
↓
로그인 여부 확인
↓
로그인 필요 시 로그인 유도
↓
공유 링크 생성
↓
친구에게 전달
```

### 7.1 친구 링크 진입

```text
친구가 링크 진입
↓
공유된 회고 읽기
↓
[나도 기억 작성하기]
```

공유 화면 문구:

```text
OO님의 기억을 보고 있어요

이 화면에서는 읽기만 할 수 있어요.
```

### 7.2 MVP 이후 답변 공개 규칙

친구 답변은 서로 답해야 공개된다.

```text
내 답변 있음 + 친구 답변 있음
↓
서로 답변 비교 공개
```

친구가 아직 답하지 않은 상태에서는 내 답변만 보여주거나, 비교 영역을 잠금 상태로 둔다.

## 8. 로그인/백업 플로우

### 8.1 로그인 필요 시점

로그인은 처음부터 요구하지 않는다.

필요한 순간:

- 안전 저장
- 백업
- 기기 동기화

### 8.2 첫 저장 이후

```text
첫 기억 작성 완료
↓
안전 저장 안내
↓
[Google로 저장하기]
[Kakao로 저장하기]
[임시로 저장하기]
```

임시 저장은 브라우저 데이터 삭제 시 사라질 수 있음을 명확히 안내한다.

### 8.3 백업

```text
프로필 메뉴
↓
백업
↓
JSON 내보내기 또는 계정 저장
```

### 8.4 가져오기

```text
프로필 메뉴
↓
가져오기
↓
JSON 파일 선택
↓
미리보기
↓
가져오기 완료
```

## 9. 프로필 메뉴 플로우

```text
상단 프로필 영역 선택
↓
메뉴 열림
↓
설정 / 언어 변경 / 테마 변경 / 가져오기 / 백업 / 로그아웃 선택
```

### 테마 변경

```text
프로필 메뉴
↓
테마 변경
↓
우주 / 방 / 지도 선택
↓
홈 장면 테마 변경
```

문구는 테마와 무관하게 중립적으로 유지한다.

## 10. MVP 구현 순서

1. 상단/하단 내비게이션 구조 정리
2. 홈 기본 진입과 기억 묶음 장면 설계
3. 첫 작성 여행 회고 7문항 wizard 구현
4. 전체 답변 검토와 대표 장면/감정 선택 구현
5. Google/Kakao 저장 유도와 임시 저장 경고 구현
6. 홈 오브젝트와 미리보기 구현
7. 다시 보기 목록/상세 구현
8. 같은 질문 비교 구현
9. 읽기 전용 공유 링크 구현
10. JSON 백업/가져오기 정리
11. 프로필 메뉴와 테마 변경 구현
