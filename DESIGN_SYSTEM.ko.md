# Recoverse v2 디자인 시스템

## 1. 디자인 방향

Recoverse는 사용자가 짧은 질문에 답하면서 자신의 기억을 발견하고, 나중에 다시 보며 과거의 감정과 생각을 재발견하는 서비스다.

디자인은 기능을 설명하는 화면보다 "내 기억들이 쌓이는 공간"을 먼저 느끼게 해야 한다.

핵심 방향:

- 조용하고 감성적인 분위기
- 세련되고 부드러운 인터랙션
- 짧고 쉬운 작성 경험
- 다시 열어보고 싶은 기억 공간
- 사용자가 고를 수 있는 테마 구조

피해야 할 방향:

- 일반적인 카드형 대시보드
- 기능 목록 중심 홈
- 긴 설명 위주의 온보딩
- SNS 피드처럼 보이는 화면
- 댓글/좋아요 중심 UI
- 과한 게임식 HUD
- 강한 네온/사이버펑크
- 복잡한 3D를 기본값으로 쓰는 구조

## 2. 제품 컨셉

테마는 여러 개를 지원할 수 있지만, 제품의 기본 감정은 하나로 유지한다.

```text
내 기억들이 쌓이고,
다시 열어보며,
나를 발견하는 공간
```

### 테마

- 기본 테마: 우주
- 추가 테마 후보: 방, 지도
- 테마 변경 위치: 상단 프로필 메뉴 > 설정/테마 변경
- 회고마다 테마를 고르게 하지 않는다.

### 문구 원칙

문구는 테마에 종속되지 않도록 중립적으로 쓴다.

사용:

- 내 기억 공간
- 기억 묶음
- 기억 조각
- 기억 작성
- 다시 보기
- 같은 질문 비교

피함:

- 기억 행성
- 그룹 은하
- 탐사 로그
- 관측 모드
- 관측 초대장

우주, 별, 궤도, 행성 같은 표현은 비주얼 테마 안에서만 사용한다.

## 3. 브랜드

브랜드는 예쁜 영문 로고와 작은 심볼을 함께 사용한다.

방향:

- 영문 로고: `Recoverse`
- 심볼: 별 + 궤도 조합
- 한글명은 메인 로고에 넣지 않는다.
- 한글 설명은 필요할 때 본문 문구에서만 사용한다.

로고 톤:

- 조용함
- 섬세함
- 너무 귀엽지 않음
- 너무 차갑지 않음
- 작은 기억 조각이 도는 듯한 느낌

## 4. 색상

색감은 테마별로 달라질 수 있다. 기본 우주 테마는 어두운 밤과 빛나는 포인트를 사용한다.

| 역할 | 토큰 | 값 | 사용처 |

| Deep Night | `space.deepNight` | `#111322` | 홈 장면 배경 |

| Memory Gold | `memory.gold` | `#F4C56A` | 주요 CTA, 빛나는 포인트 |
| Warm Peach | `memory.peach` | `#F2A27E` | 따뜻한 강조 |

색상 규칙:

- 한 화면이 보라색만으로 보이지 않게 한다.
- 금색은 중요한 행동과 기억의 빛에만 쓴다.
- 청록은 보조 연결감과 상태 표현에만 쓴다.
- 위험 액션은 별도 경고색을 사용한다.
- 형광 네온, 강한 사이버펑크 핑크, 과한 블루는 사용하지 않는다.

## 5. 타이포그래피

제품 언어는 짧고 담백해야 한다.

원칙:

- 사용자가 글을 쓰는 동안 문구가 앞서 나가지 않는다.
- 기능 설명보다 행동을 짧게 안내한다.
- 감성 문구는 완료/빈 상태/다시 보기에서만 절제해 쓴다.
- 메뉴명은 명확하게 유지한다.

주요 메뉴:

```text
기억 작성 / 홈 / 다시 보기
```

크기 규칙:

| 역할 | 모바일 | PC | 규칙 |
| --- | --- | --- | --- |
| 브랜드 | 18-22px | 20-26px | 상단에서 또렷하게 |
| 화면 제목 | 24-30px | 30-40px | 짧고 감정 있게 |
| 질문 | 22-28px | 28-36px | 작성 화면의 중심 |
| 섹션 제목 | 17-20px | 20-24px | 목록/패널 제목 |
| 본문 | 15-17px | 16-18px | 답변 읽기 우선 |
| 보조 정보 | 12-14px | 12-14px | 날짜, 진행률, 상태 |

규칙:

- 글자 간격은 기본값을 사용한다.
- 모바일에서 제목은 자연스럽게 줄바꿈한다.
- 버튼 문구는 짧게 유지한다.
- 홈에서는 긴 설명 문장을 노출하지 않는다.

## 6. 내비게이션

### 상단

상단은 브랜드와 프로필 메뉴만 둔다.

```text
Recoverse 로고 / 프로필 영역
```

프로필 메뉴에 포함:

- 설정
- 언어 변경
- 테마 변경
- 가져오기
- 백업
- 로그아웃

### 하단

하단 탭은 세 개만 둔다.

```text
기억 작성 / 홈 / 다시 보기
```

기본 진입은 홈이다.

작성 중 하단 탭을 누르면 확인한다.

```text
작성 중인 기억이 있어요.
이동하면 이어서 작성할 수 있어요.
```

## 7. 홈

홈은 기능 목록이 아니라 기억이 쌓이는 인터랙티브 장면이다.

목표:

- 내 기억이 쌓이고 있다는 감각
- 시간순 구조 이해
- 오브젝트를 눌러 탐색하는 재미

기본 구조:

```text
상단 브랜드
기억 공간 장면
  ├─ 연도별 기억 묶음
  ├─ 기억 오브젝트
  └─ 미리보기 패널
하단 탭
```

홈에서 보여줄 것:

- 연도별 기억 묶음
- 같은 해의 올해/상반기/하반기 묶음
- 오브젝트 형태의 기억

예:

```text
2026
- 2026년의 나
- 2026 상반기의 나
- 2026 하반기의 나
```

홈에서 하지 않는 것:

- 긴 서비스 설명
- 복잡한 통계
- 공유 CTA 전면 노출
- JSON 관리
- 전체 삭제 같은 위험 액션

## 8. 홈 인터랙션

MVP 기본:

```text
오브젝트 클릭
↓
상세 미리보기 패널
↓
[열어보기]
↓
다시 보기 상세
```

확장:

- 오브젝트 드래그
- 공간 회전
- 주변 탐험
- 테마별 공간 스크롤

MVP에서는 길을 잃지 않도록 클릭과 미리보기를 먼저 완성한다.

## 9. 오브젝트 스타일

MVP 오브젝트는 미니멀한 점/원/궤도 중심으로 만든다.

기본 우주 테마:

- 작은 별점
- 원형 기억 묶음
- 은은한 궤도
- 부드러운 움직임
- 선택된 기억의 약한 빛

나중에 확장:

- 일러스트 행성/별/장식
- 3D/입체 오브젝트
- 테마별 오브젝트 세트

규칙:

- 기본 화면에서 오브젝트가 글을 방해하면 안 된다.
- 클릭 가능한 오브젝트는 명확한 hover/pressed 상태를 가진다.
- `prefers-reduced-motion`에서는 움직임을 줄인다.

## 10. 기억 작성 화면

작성 화면은 카드형 질문과 완성되는 장면을 결합한다.

확정 방향:

```text
질문 카드형 + 답할수록 장면이 완성됨
```

기본 화면:

```text
3 / 8

[질문 카드]
올해 가장 먼저 떠오르는 순간은?

[짧게 적어도 괜찮아요]

+ 더 길게 쓰기

[이전] [다음]
```

질문 카드 전환:

- 좌우 슬라이드
- 짧고 부드럽게
- 과한 튕김 없음
- 글쓰기를 방해하지 않는 속도

답변 입력:

- 기본은 짧은 문장
- 길게 쓰기는 선택
- 태그/선택지는 보조

## 11. 완료 화면

완료 화면은 기억이 저장되는 감각을 준다.

포함:

- 기억이 완성됐다는 문구
- 내 기억 공간에 저장되는 연출
- 로그인하고 안전하게 저장 CTA
- 나중에 하기
- 친구에게 같은 질문 보내기 작은 제안

구조:

```text
기억이 완성됐어요.

내 기억 공간에 저장할게요.

[로그인하고 안전하게 저장]
[나중에 하기]

친구들은 이 질문에 어떻게 답할까요?
[친구에게 같은 질문 보내기]
```

## 12. 다시 보기

다시 보기는 사용자가 작성한 기억 묶음을 열고, 같은 질문의 답변을 비교하는 공간이다.

기본 구조:

```text
2026
[2026년의 나]
[2026 상반기의 나]
[2026 하반기의 나]

여행
[제주 여행의 기억]
```

상세 구조:

```text
2026 상반기의 나

1. 가장 먼저 떠오르는 순간은?
답변...

2. 그때 가장 많이 느낀 감정은?
답변...

[같은 질문 비교]
[공유하기]
```

같은 질문 비교는 회고의 핵심 가치다.

```text
질문
가장 먼저 떠오르는 순간은?

2026 상반기
답변...

2026 하반기
답변...

2026년
답변...
```

## 13. 공유 화면

공유 화면은 읽기와 같은 질문 답변이 목적이다.

친구가 할 수 있는 것:

- 공유된 회고 읽기
- 같은 질문에 답하기

친구가 할 수 없는 것:

- 원본 수정
- 삭제
- 작성자 데이터 관리

공개 규칙:

```text
서로 답해야 비교 공개
```

## 14. 버튼

### 주요 버튼

사용처:

- 기억 작성 시작
- 다음 질문
- 저장
- 로그인하고 안전하게 저장
- 열어보기

규칙:

- 모바일 터치 높이 44px 이상
- 한 화면의 주요 CTA는 하나만 선명하게
- 기본 우주 테마에서는 `Memory Gold`를 중심으로 사용

### 보조 버튼

사용처:

- 나중에 하기
- 더 길게 쓰기
- 공유하기
- 가져오기/백업

규칙:

- 주요 CTA보다 낮은 대비
- 위치와 목적을 명확히
- 홈 장면을 방해하지 않음

### 위험 버튼

사용처:

- 기억 삭제
- 전체 데이터 삭제
- 로그아웃

규칙:

- 홈 첫 화면에 노출하지 않는다.
- 프로필 메뉴나 상세 하위 영역에 둔다.
- 확인 절차를 둔다.

## 15. 여백과 레이아웃

모바일 우선으로 설계한다.

| 역할 | 값 |
| --- | --- |
| 화면 좌우 padding | 18-20px |
| 섹션 간격 | 28-40px |
| 카드 내부 padding | 18-22px |
| 하단 탭 높이 | 64-76px |
| 터치 대상 최소 크기 | 44px |

규칙:

- 홈 장면은 충분한 숨 쉴 공간을 가진다.
- 질문 카드는 화면 중앙 집중을 유지한다.
- 하단 CTA는 키보드와 겹치지 않아야 한다.
- 모바일에서 텍스트가 버튼 안에서 줄바꿈/잘림되지 않아야 한다.

## 16. 애니메이션

허용:

- 별의 미세한 opacity 변화
- 오브젝트의 느린 부유
- 오브젝트 클릭 시 짧은 확대
- 질문 카드 좌우 슬라이드
- 완료 시 오브젝트가 공간에 놓이는 연출
- 미리보기 패널의 부드러운 등장

금지:

- 빠른 파티클 폭발
- 강한 네온 깜빡임
- 글쓰기 중 반복되는 큰 움직임
- 게임식 HUD
- 읽기를 방해하는 배경 애니메이션

권장 값:

```text
object float: 6-10s ease-in-out infinite
star twinkle: 3-6s ease-in-out infinite
card slide: 220-320ms ease
tap scale: 160-220ms ease-out
panel reveal: 240-320ms ease
screen transition: 240-360ms ease
```

## 17. 접근성

- 모든 주요 버튼은 키보드 포커스 상태를 가진다.
- 애니메이션은 `prefers-reduced-motion`을 따른다.
- 색상만으로 상태를 구분하지 않는다.
- 오브젝트 클릭 영역은 최소 44px 이상으로 만든다.
- 공유/읽기 화면은 읽기 순서가 자연스러워야 한다.
- 입력 화면에서는 질문, 입력창, 다음 버튼 순서가 명확해야 한다.

## 18. 구현 우선순위

1. 상단 브랜드/프로필 메뉴와 하단 탭 정리
2. 기본 우주 테마 색상과 문구 체계 정리
3. 홈 기억 공간 장면과 오브젝트 미리보기 구현
4. 기억 작성 카드형 wizard 구현
5. 질문 카드 좌우 슬라이드 구현
6. 완료 화면과 저장 연출 구현
7. 다시 보기와 같은 질문 비교 화면 정리
8. 공유 읽기/같은 질문 답변 화면 정리
9. 테마 변경 구조 준비
