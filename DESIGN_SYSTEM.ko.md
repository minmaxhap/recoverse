# Recoverse 디자인 시스템

## 디자인 방향

Recoverse는 기억을 탐험하는 우주 아카이브 앱이다.

홈 화면은 사용자의 캡슐이 행성처럼 떠 있는 기억 지도여야 한다. 첫인상은 생산성 도구나 일반 회고 앱이 아니라, 조용하고 따뜻한 개인 우주에 들어온 느낌이어야 한다.

핵심 시각 키워드:

- 우주 아카이브
- 기억 행성
- 따뜻한 밤하늘
- 작은 탐험
- 발견
- 자기이해
- 우주 정원

피해야 할 방향:

- SF 게임
- 사이버펑크
- 차가운 네온
- 복잡한 3D
- 전투적이거나 기계적인 HUD
- 일반 메모앱처럼 납작한 목록 화면

## 색상

기본 배경은 깊은 우주색을 사용하되, 포인트는 따뜻한 금빛, 복숭아빛, 라벤더빛으로 잡는다.

| 역할 | 토큰 | 값 | 사용처 |
| --- | --- | --- | --- |
| Space Black | `space.black` | `#08070F` | 전체 배경 |
| Warm Night | `space.warmNight` | `#15111F` | 패널 배경, 깊이감 |
| Archive Navy | `space.archiveNavy` | `#1D2438` | 보조 배경 |
| Dust Purple | `space.dustPurple` | `#6D5A8D` | 궤도선, 보조 포인트 |
| Soft Lavender | `space.softLavender` | `#B9A7E8` | 선택 상태, 작은 별 |
| Memory Gold | `memory.gold` | `#F4C56A` | 주요 CTA, 발견 상태 |
| Sunset Peach | `memory.peach` | `#F2A27E` | 따뜻한 행성 표면 |
| Rose Fog | `memory.roseFog` | `#E8A8B8` | 감성 강조 |
| Moon Paper | `text.moonPaper` | `#F6EEDC` | 주요 텍스트 |
| Star White | `text.starWhite` | `#FFF9EA` | 제목, 강조 텍스트 |
| Muted Star | `text.mutedStar` | `#BDB4C8` | 보조 텍스트 |

사용 규칙:

- 한 화면이 보라색 계열로만 보이지 않게 금빛, 복숭아빛, 남색을 함께 사용한다.
- 네온 블루, 형광 그린, 강한 사이버펑크 핑크는 사용하지 않는다.
- 배경은 어둡지만, 감정은 차갑지 않아야 한다.
- 중요한 액션은 `Memory Gold` 또는 `Sunset Peach`를 사용한다.

## 타이포그래피

한국어가 기본이다. 영어는 설정에서 선택한 경우에만 표시한다.

권장 톤:

- 짧고 감성적인 문장
- 기능 설명보다 발견의 언어
- 사용자를 재촉하지 않는 문장

예시 문구:

- `나의 기억 우주`
- `오늘 다시 발견한 기억`
- `새 기억 행성 만들기`
- `이 캡슐을 다시 열어볼까요?`
- `아직 답변하지 않은 기억 카드`

크기 규칙:

| 역할 | 모바일 | PC | 규칙 |
| --- | --- | --- | --- |
| 화면 제목 | 24-28px | 28-36px | 홈의 정체성을 만든다 |
| 섹션 제목 | 17-20px | 20-24px | 짧고 명확하게 |
| 카드 제목 | 15-18px | 16-20px | 두 줄 이내 |
| 본문 | 14-16px | 15-17px | 긴 답변 가독성 우선 |
| 보조 정보 | 12-13px | 12-14px | 날짜, 진행률, 유형 |

타이포그래피 규칙:

- 글자 간격은 기본값을 사용한다.
- 모바일에서 제목이 길면 줄바꿈을 허용한다.
- 버튼 안 텍스트는 한 줄을 우선하되, 좁은 화면에서는 아이콘+짧은 라벨을 사용한다.
- 장식적인 폰트는 로고나 짧은 감성 문구에만 제한한다.

## 버튼 규칙

버튼은 명령을 명확히 보여주되, 우주 콘셉트를 과장하지 않는다.

### 주요 버튼

사용처:

- 새 기억 행성 만들기
- 답변 저장
- 다시 열어보기

규칙:

- 배경은 `Memory Gold` 또는 `Sunset Peach`.
- 텍스트는 `#08070F` 계열의 어두운 색을 사용한다.
- 모바일 기준 높이는 44px 이상.
- 둥근 정도는 999px pill 또는 12-16px radius를 사용한다.

### 보조 버튼

사용처:

- 가져오기
- 내보내기
- 필터
- 새로고침

규칙:

- 투명 배경 또는 어두운 반투명 배경.
- 얇은 라인과 낮은 대비의 별빛 효과를 사용한다.
- 아이콘이 있는 경우 라벨은 짧게 유지한다.

### 위험 버튼

사용처:

- 캡슐 삭제
- 질문 카드 삭제

규칙:

- 우주 콘셉트와 섞지 않는다.
- 명확한 경고 색과 확인 절차를 사용한다.
- 홈 첫 화면에는 노출하지 않는다.

## 카드 규칙

일반 카드는 정보를 담는 표면이다. 홈에서는 카드가 화면을 지배하지 않아야 하며, 우주 지도와 행성 캡슐이 주인공이다.

공통 규칙:

- 반투명 어두운 배경을 사용한다.
- 테두리는 낮은 대비의 라벤더/금빛 라인.
- 그림자는 강하게 띄우지 않고, 내부 glow를 약하게 준다.
- radius는 16-24px 범위.
- 카드 안에 카드가 중첩되지 않게 한다.

오늘의 발견 카드:

- 홈 상단의 감정적 진입점이다.
- 오래된 답변 한 조각, 캡슐 이름, 다시 열기 액션을 포함한다.
- 큰 설명문보다 짧은 인용문이 중심이다.
- 작은 별, 꽃잎, 미세한 행성 장식은 허용한다.

내 캡슐 목록:

- 지도 아래 보조 영역으로 둔다.
- 목록은 관리 도구처럼 보이지 않게 행성 아이콘, 진행률, 최근 수정일을 함께 사용한다.
- 모바일에서는 한 줄 리스트, PC에서는 compact card grid를 허용한다.

## 행성 카드 규칙

행성 카드는 Recoverse 홈의 핵심 컴포넌트다.

의미:

```text
행성 = 캡슐
행성의 밝기/고리/별 = 기록 상태
행성 클릭 = 캡슐 탐험 시작
```

구성 요소:

- 행성 원형 비주얼
- 캡슐 제목
- 캡슐 유형
- 질문 수 / 답변 수
- 최근 수정 상태

시각 규칙:

- CSS gradient만으로 행성 표면을 표현한다.
- Three.js나 복잡한 canvas 렌더링을 사용하지 않는다.
- 행성마다 색은 다르지만 전체 팔레트 안에서 유지한다.
- 고리는 `border`, `pseudo-element`, `transform: rotate()`로 표현한다.
- 캡슐 타입별로 표면 색을 약하게 다르게 줄 수 있다.

타입별 방향:

| 타입 | 시각 방향 |
| --- | --- |
| 연도 | 금빛/복숭아빛 행성 |
| 여행 | 청록/라벤더 행성 |
| 프로젝트 | 남색/금빛 고리 |
| 인간관계 | 장밋빛/보랏빛 행성 |
| 커리어 | 차분한 남색/회색빛 행성 |
| 특정 시기 | 부드러운 라벤더/달빛 행성 |

상태 표현:

- 답변이 많을수록 행성이 조금 더 밝다.
- 미답변 카드가 많으면 고리가 흐리다.
- 최근 수정된 캡슐은 작은 별 하나가 붙는다.
- 오늘의 발견 대상 캡슐은 약한 금빛 halo를 가진다.

상호작용:

- 탭/클릭 시 `scale(1.06)` 정도로 확대한다.
- 확대 후 캡슐 상세로 전환한다.
- hover가 없는 모바일에서는 press 상태를 짧게 보여준다.
- 애니메이션은 느리고 부드러워야 한다.

## 여백 규칙

모바일 우선으로 설계한다.

기본 간격:

| 역할 | 값 |
| --- | --- |
| 화면 좌우 padding | 20px |
| 섹션 간격 | 28-40px |
| 카드 내부 padding | 16-20px |
| 목록 아이템 간격 | 10-14px |
| 행성 지도 최소 높이 | 360px |
| 터치 대상 최소 크기 | 44px |

홈 배치:

```text
상단 브랜드
오늘의 발견 카드
기억 지도
새 기억 행성 만들기
내 캡슐 목록
```

규칙:

- 지도 영역은 숨 쉬는 공간이 있어야 한다.
- 행성은 일정한 grid보다 비대칭 배치를 우선한다.
- 모바일에서는 첫 화면에서 오늘의 발견과 기억 지도 시작 부분이 보여야 한다.
- PC에서는 지도 영역을 넓히고, 보조 패널은 한쪽에 배치할 수 있다.

## 애니메이션 규칙

애니메이션은 “탐험”과 “발견”을 보조해야 한다.

허용:

- 행성의 느린 부유 애니메이션
- 별의 미세한 opacity 변화
- 캡슐 클릭 시 짧은 확대
- 카드가 아래에서 부드럽게 올라오는 전환
- 오늘의 발견 카드의 약한 glow

금지:

- 빠른 파티클 폭발
- 강한 네온 깜빡임
- 3D 회전
- 게임식 전투 HUD
- 사용자의 읽기를 방해하는 반복 애니메이션

권장 값:

```text
planet float: 6-10s ease-in-out infinite
star twinkle: 3-6s ease-in-out infinite
tap scale: 160-220ms ease-out
screen transition: 240-360ms ease
```

접근성:

- `prefers-reduced-motion`에서는 부유와 twinkle을 끈다.
- 중요한 정보는 애니메이션에만 의존하지 않는다.

## 다크모드 방향

Recoverse의 기본 모드는 다크모드다.

이유:

- 우주 아카이브 콘셉트와 맞다.
- 행성, 별, 발견 카드가 잘 드러난다.
- 밤에 회고를 작성하는 사용 맥락과 잘 맞다.

다크모드 규칙:

- 배경은 거의 검정에 가까운 따뜻한 색을 사용한다.
- 텍스트는 완전한 흰색보다 `Moon Paper` 계열을 우선한다.
- 카드와 패널은 배경보다 한 단계 밝게 한다.
- 주요 CTA는 어두운 배경 위에서 충분한 대비를 가져야 한다.

라이트모드 방향:

- MVP에서는 필수 구현 범위가 아니다.
- 나중에 추가할 경우 “낮의 아카이브”보다 “종이 지도” 느낌으로 설계한다.
- 라이트모드에서도 행성 지도 콘셉트는 유지한다.
