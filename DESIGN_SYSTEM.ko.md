# Recoverse 디자인 시스템

## 디자인 방향

Recoverse는 기억을 탐험하는 우주 아카이브 서비스다.

앱의 첫 화면은 목록이나 관리 도구가 아니라 사용자의 기억 우주여야 한다. 개인 회고는 행성, 그룹 회고는 은하, 질문 카드는 탐사 기록/별, 답변은 탐사 로그로 표현한다.

핵심 시각 키워드:

- 우주 아카이브
- 기억 행성
- 그룹 은하
- 별자리
- 관측 모드
- 따뜻한 밤하늘
- 조용한 탐험
- 자기이해와 발견

피해야 할 방향:

- 일반적인 카드형 대시보드
- 캡슐 목록 중심 홈
- 상단 탭 중심 구조
- SF 게임
- 사이버펑크
- 차가운 네온
- 복잡한 3D
- SNS 피드
- 댓글/좋아요 중심 UI

## 색상

기본 배경은 깊은 우주색을 사용하되, 감정은 차갑지 않게 금빛, 복숭아빛, 라벤더빛을 섞는다.

| 역할 | 토큰 | 값 | 사용처 |
| --- | --- | --- | --- |
| Space Black | `space.black` | `#08070F` | 전체 배경 |
| Warm Night | `space.warmNight` | `#15111F` | 우주 표면, 깊이감 |
| Archive Navy | `space.archiveNavy` | `#1D2438` | 보조 배경 |
| Dust Purple | `space.dustPurple` | `#6D5A8D` | 궤도선, 보조 포인트 |
| Soft Lavender | `space.softLavender` | `#B9A7E8` | 선택 상태, 작은 별 |
| Memory Gold | `memory.gold` | `#F4C56A` | 오늘의 발견, 주요 CTA |
| Sunset Peach | `memory.peach` | `#F2A27E` | 따뜻한 행성 표면 |
| Rose Fog | `memory.roseFog` | `#E8A8B8` | 감성 강조 |
| Galaxy Teal | `galaxy.teal` | `#60D0A8` | 그룹 은하, 멤버 상태 |
| Moon Paper | `text.moonPaper` | `#F6EEDC` | 본문 텍스트 |
| Star White | `text.starWhite` | `#FFF9EA` | 제목, 강조 텍스트 |
| Muted Star | `text.mutedStar` | `#BDB4C8` | 보조 텍스트 |

사용 규칙:

- 한 화면이 보라색만으로 보이지 않게 금빛, 복숭아빛, 남색, 청록을 섞는다.
- 그룹 은하는 개인 행성과 구분되도록 청록/녹색 계열을 약하게 사용한다.
- 네온 블루, 형광 그린, 강한 사이버펑크 핑크는 사용하지 않는다.
- 위험 액션은 우주 색상에 숨기지 않고 명확한 경고 색을 사용한다.

## 타이포그래피

한국어가 기본이다. 영어는 설정에서 선택한 경우에만 표시한다.

제품 언어는 기능 설명보다 탐험의 언어를 우선한다.

| 기존 표현 | 새 표현 |
| --- | --- |
| 캡슐 | 개인 행성 / 기억 행성 |
| 그룹 회고 | 은하 |
| 질문 카드 | 탐사 기록 / 별 |
| 답변 | 탐사 로그 |
| 공유 링크 | 관측 초대장 |
| 공유 화면 | 관측 모드 |

예시 문구:

- `나의 기억 우주`
- `오늘 다시 발견한 별`
- `새 기억 행성 만들기`
- `그룹 은하 만들기`
- `관측 초대장 만들기`
- `이 행성을 다시 탐험할까요?`

크기 규칙:

| 역할 | 모바일 | PC | 규칙 |
| --- | --- | --- | --- |
| 화면 제목 | 24-28px | 28-36px | 홈의 정체성을 만든다 |
| 섹션 제목 | 17-20px | 20-24px | 짧고 명확하게 |
| 오브젝트 라벨 | 12-14px | 13-15px | 행성/은하 이름 |
| 본문 | 14-16px | 15-17px | 긴 탐사 로그 가독성 우선 |
| 보조 정보 | 12-13px | 12-14px | 날짜, 진행률, 유형 |

규칙:

- 글자 간격은 기본값을 사용한다.
- 모바일에서 제목이 길면 줄바꿈을 허용한다.
- 버튼 문구는 짧게 유지한다.
- 홈에서는 기능 설명 문장을 길게 노출하지 않는다.

## 홈 우주 뷰 규칙

홈은 관리 화면이 아니라 우주 공간이다.

포함:

- 상단의 작은 브랜드/상태 영역
- 오늘의 발견
- 개인 행성
- 그룹 은하
- `+` 생성 진입점

제외:

- 캡슐 목록 중심 영역
- 연도 보기
- 빠른 입력
- 질문 비교
- JSON 관리
- 전체 삭제 같은 위험 액션

배치:

```text
상단 브랜드
오늘의 발견
우주 지도
  ├─ 개인 행성
  ├─ 그룹 은하
  └─ + 생성 진입점
하단 내비게이션 또는 아카이브 진입
```

모바일 첫 화면에서는 오늘의 발견과 우주 지도 시작 부분이 함께 보여야 한다.

## 버튼 규칙

### 주요 버튼

사용처:

- 새 기억 행성 만들기
- 그룹 은하 만들기
- 탐사 로그 저장
- 다시 탐험하기

규칙:

- 배경은 `Memory Gold`, `Sunset Peach`, 또는 그룹 기능의 경우 `Galaxy Teal`.
- 모바일 기준 높이는 44px 이상.
- 홈의 `+` 버튼은 단순 추가가 아니라 생성 선택지를 여는 탐험 진입점이다.

### 보조 버튼

사용처:

- 아카이브 보기
- 설정
- 필터
- 가져오기/내보내기

규칙:

- 홈의 중심부가 아니라 보조 위치에 둔다.
- 투명 또는 어두운 반투명 배경을 사용한다.
- 라벨은 짧게 유지한다.

### 위험 버튼

사용처:

- 행성 삭제
- 탐사 기록 삭제
- 전체 데이터 삭제

규칙:

- 홈 첫 화면에 노출하지 않는다.
- 아카이브/설정 또는 상세 화면 내부에서 확인 절차와 함께 사용한다.

## 카드 규칙

카드는 정보를 담는 표면이다. 홈에서는 카드가 우주 지도를 압도하면 안 된다.

오늘의 발견 카드:

- 홈의 감정적 진입점이다.
- 오래된 탐사 로그 한 조각과 연결된 행성/별을 보여준다.
- 길게 설명하지 않고 짧은 문장과 다시 열기 액션을 둔다.

아카이브 카드:

- 목록, 필터, JSON 관리 같은 기능은 아카이브에서 카드 형태로 정리한다.
- 홈으로 역류하지 않게 한다.

관측 모드 카드:

- 읽기 전용임을 명확히 표시한다.
- 편집, 댓글, 좋아요 액션을 두지 않는다.

## 행성 규칙

의미:

```text
행성 = 개인 회고 캡슐
행성 클릭 = 행성 상세 탐험
행성 밝기/고리/별 = 기록 상태
```

시각 규칙:

- CSS gradient만으로 행성 표면을 표현한다.
- Three.js나 복잡한 canvas 렌더링을 사용하지 않는다.
- 캡슐 타입별로 행성 색과 고리를 약하게 다르게 준다.
- 최근 수정된 행성에는 작은 별을 붙인다.
- 오늘의 발견 대상 행성에는 약한 금빛 halo를 준다.

상호작용:

- 탭/클릭 시 `scale(1.06)` 정도로 확대한다.
- 확대 후 행성 상세로 전환한다.
- `prefers-reduced-motion`에서는 확대/부유 애니메이션을 줄인다.

## 은하 규칙

의미:

```text
은하 = 그룹 회고
중심 코어 = 그룹 주제
위성 행성 = 참여자
별자리 = 공통 탐사 기록
```

시각 규칙:

- 개인 행성보다 더 큰 궤도 구조를 가진다.
- 청록/라벤더 계열을 사용해 개인 행성과 구분한다.
- 멤버는 작은 위성 행성으로 표현한다.
- 아직 실제 그룹 기능이 없을 때는 생성 진입점과 빈 상태만 준비한다.

## 관측 모드 규칙

관측 모드는 공유 읽기 전용 화면이다.

규칙:

- 상단에 `관측 모드`와 읽기 전용 상태를 명확히 표시한다.
- 편집 버튼, 삭제 버튼, 입력 필드를 노출하지 않는다.
- 원본 앱보다 조용하고 감상 중심으로 구성한다.
- 공유 링크는 `관측 초대장`이라고 부른다.

## 여백과 레이아웃

모바일 우선으로 설계한다.

| 역할 | 값 |
| --- | --- |
| 화면 좌우 padding | 20px |
| 섹션 간격 | 28-40px |
| 카드 내부 padding | 16-20px |
| 우주 지도 최소 높이 | 420px |
| 터치 대상 최소 크기 | 44px |

규칙:

- 홈 지도 영역은 충분한 숨 쉴 공간을 가져야 한다.
- 행성/은하는 비대칭 배치를 우선한다.
- PC에서는 우주 지도를 넓게 두고 보조 기능은 아카이브/사이드 영역으로 분리한다.

## 애니메이션 규칙

허용:

- 행성의 느린 부유
- 별의 미세한 opacity 변화
- 행성/은하 클릭 시 짧은 확대
- 관측 모드 진입 시 부드러운 fade

금지:

- 빠른 파티클 폭발
- 강한 네온 깜빡임
- 3D 회전
- 게임식 HUD
- 읽기를 방해하는 반복 애니메이션

권장 값:

```text
planet float: 6-10s ease-in-out infinite
star twinkle: 3-6s ease-in-out infinite
tap scale: 160-220ms ease-out
screen transition: 240-360ms ease
```

## 다크모드 방향

Recoverse는 다크모드 우선이다.

- 우주 아카이브 콘셉트와 맞다.
- 행성, 은하, 별, 관측 모드가 잘 드러난다.
- 밤에 회고를 작성하는 맥락과 잘 맞다.

라이트모드는 MVP 필수 범위가 아니다. 나중에 추가한다면 “낮의 대시보드”가 아니라 “종이 별지도” 느낌으로 설계한다.
