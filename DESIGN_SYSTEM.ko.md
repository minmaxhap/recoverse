# Recoverse v2 디자인 시스템

## 1. 디자인 방향

Recoverse는 사용자가 짧은 질문에 답하면서 자신의 기억을 발견하고, 나중에 다시 보며 과거의 감정과 생각을 재발견하는 서비스다.

디자인은 기능을 설명하는 화면보다 "내 기억들이 쌓이는 공간"을 먼저 느끼게 해야 한다.

핵심 방향:

- 조용하고 감성적인 분위기
- 세련되고 부드러운 인터랙션
- 짧고 쉬운 작성 경험
- 다시 열어보고 싶은 기억 공간
- 사용자가 고를 수 있는 테마 구조

피해야 할 방향:

- 일반적인 카드형 대시보드
- 기능 목록 중심 홈
- 긴 설명 위주의 온보딩
- SNS 피드처럼 보이는 화면
- 댓글/좋아요 중심 UI
- 과한 게임식 HUD
- 강한 네온/사이버펑크
- 복잡한 3D를 기본값으로 쓰는 구조

## 2. 제품 컨셉

테마는 여러 개를 지원할 수 있지만, 제품의 기본 감정은 하나로 유지한다.

```text
내 기억들이 쌓이고,
다시 열어보며,
나를 발견하는 공간
```

### 테마

- 기본 테마: 우주
- 추가 테마 후보: 방, 지도
- 테마 변경 위치: 상단 프로필 메뉴 > 설정/테마 변경
- 회고마다 테마를 고르게 하지 않는다.

### 문구 원칙

문구는 테마에 종속되지 않도록 중립적으로 쓴다.

사용:

- 내 기억 공간
- 기억 묶음
- 기억 조각
- 기억 작성
- 다시 보기
- 같은 질문 비교

피함:

- 기억 행성
- 그룹 은하
- 탐사 로그
- 관측 모드
- 관측 초대장

우주, 별, 궤도, 행성 같은 표현은 비주얼 테마 안에서만 사용한다.

## 3. 브랜드

브랜드는 예쁜 영문 로고와 작은 심볼을 함께 사용한다.

방향:

- 영문 로고: `Recoverse`
- 심볼: 별 + 궤도 조합
- 한글명은 메인 로고에 넣지 않는다.
- 한글 설명은 필요할 때 본문 문구에서만 사용한다.

로고 톤:

- 조용함
- 섬세함
- 너무 귀엽지 않음
- 너무 차갑지 않음
- 작은 기억 조각이 도는 듯한 느낌

- 한 화면이 보라색만으로 보이지 않게 한다.
- 금색은 중요한 행동과 기억의 빛에만 쓴다.
- 청록은 보조 연결감과 상태 표현에만 쓴다.
- 위험 액션은 별도 경고색을 사용한다.
- 형광 네온, 강한 사이버펑크 핑크, 과한 블루는 사용하지 않는다.

## 5. 타이포그래피

제품 언어는 짧고 담백해야 한다.

원칙:

- 사용자가 글을 쓰는 동안 문구가 앞서 나가지 않는다.
- 기능 설명보다 행동을 짧게 안내한다.
- 감성 문구는 완료/빈 상태/다시 보기에서만 절제해 쓴다.
- 메뉴명은 명확하게 유지한다.

주요 메뉴:

```text
기억 작성 / 홈 / 다시 보기
```

크기 규칙:

| 역할 | 모바일 | PC | 규칙 |
| --- | --- | --- | --- |
| 브랜드 | 18-22px | 20-26px | 상단에서 또렷하게 |
| 화면 제목 | 24-30px | 30-40px | 짧고 감정 있게 |
| 질문 | 22-28px | 28-36px | 작성 화면의 중심 |
| 섹션 제목 | 17-20px | 20-24px | 목록/패널 제목 |
| 본문 | 15-17px | 16-18px | 답변 읽기 우선 |
| 보조 정보 | 12-14px | 12-14px | 날짜, 진행률, 상태 |

규칙:

- 글자 간격은 기본값을 사용한다.
- 모바일에서 제목은 자연스럽게 줄바꿈한다.
- 버튼 문구는 짧게 유지한다.
- 홈에서는 긴 설명 문장을 노출하지 않는다.

## 6. 내비게이션

### 상단

상단은 브랜드와 프로필 메뉴만 둔다.

```text
Recoverse 로고 / 프로필 영역
```

프로필 메뉴에 포함:

- 설정
- 언어 변경
- 테마 변경
- 가져오기
- 백업
- 로그아웃

### 하단

하단 탭은 세 개만 둔다.

```text
기억 작성 / 홈 / 다시 보기
```

기본 진입은 홈이다.

작성 중 하단 탭을 누르면 확인한다.

```text
작성 중인 기억이 있어요.
이동하면 이어서 작성할 수 있어요.
```

## 7. 홈

홈은 기능 목록이 아니라 기억이 쌓이는 인터랙티브 장면이다.

목표:

- 내 기억이 쌓이고 있다는 감각
- 시간순 구조 이해
- 오브젝트를 눌러 탐색하는 재미

첫 진입/빈 상태에서는 짧은 제품 설명과 첫 작성 CTA만 보여준다.

```text
내 기억이 쌓이는 공간

짧은 질문에 답하면
오늘의 생각과 감정이 하나의 기억으로 남아요.

나중에 다시 열어보면
그때의 나를 조금 다르게 발견할 수 있어요.

[첫 기억 작성하기]
```

기본 구조:

```text
상단 브랜드
기억 공간 장면
  ├─ 연도별 기억 묶음
  ├─ 기억 오브젝트
  └─ 미리보기 패널
하단 탭
```

홈에서 보여줄 것:

- 연도별 기억 묶음
- 같은 해의 올해/상반기/하반기 묶음
- 오브젝트 형태의 기억

예:

```text
2026
- 2026년의 나
- 2026 상반기의 나
- 2026 하반기의 나
```

홈에서 하지 않는 것:

- 긴 기능 설명
- 복잡한 통계
- 공유 CTA 전면 노출
- JSON 관리
- 전체 삭제 같은 위험 액션

## 8. 홈 인터랙션

MVP 기본:

```text
오브젝트 클릭
↓
제목/대표 감정/대표 장면 미리보기
↓
[열어보기]
↓
다시 보기 상세
```

미리보기 예:

```text
제주 여행

자유로움

"바다 앞에서 먹은 라면"

[열어보기]
```

확장:

- 여행/연도/20대별 감성 오브젝트
- 오브젝트 드래그
- 공간 회전
- 주변 탐험
- 테마별 공간 스크롤

MVP에서는 길을 잃지 않도록 단순 원/별 오브젝트, 클릭, 미리보기를 먼저 완성한다.

## 9. 오브젝트 스타일

MVP 오브젝트는 미니멀한 점/원/궤도 중심으로 만든다.

기본 우주 테마:

- 작은 별점
- 원형 기억 묶음
- 은은한 궤도
- 부드러운 움직임
- 선택된 기억의 약한 빛

나중에 확장:

- 일러스트 행성/별/장식
- 3D/입체 오브젝트
- 테마별 오브젝트 세트

규칙:

- 기본 화면에서 오브젝트가 글을 방해하면 안 된다.
- 클릭 가능한 오브젝트는 명확한 hover/pressed 상태를 가진다.
- `prefers-reduced-motion`에서는 움직임을 줄인다.

## 10. 기억 작성 화면

작성 화면은 카드형 질문과 완성되는 장면을 결합한다.

확정 방향:

```text
한 질문씩 카드로 답변
↓
마지막 전체 검토
↓
대표 장면/감정 선택
```

기본 화면:

```text
1 / 7

[질문 카드]
그 여행에서 가장 먼저 떠오르는 장면은?

[짧게 적어도 괜찮아요]

+ 더 길게 쓰기

[이전] [다음]
```

질문 카드 전환:

- 좌우 슬라이드
- 짧고 부드럽게
- 과한 튕김 없음
- 글쓰기를 방해하지 않는 속도

답변 입력:

- 기본은 짧은 문장
- 길게 쓰기는 선택
- 태그/선택지는 보조

첫 작성의 대표 케이스는 여행이며 기본 질문은 7개다.

여행 기본 질문:

1. 그 여행에서 가장 먼저 떠오르는 장면은?
2. 그때 가장 많이 느낀 감정은?
3. 같이 있던 사람이나 떠오르는 얼굴은?
4. 계속 생각나는 음식, 노래, 냄새가 있다면?
5. 예상과 달랐던 순간은?
6. 그 여행이 끝난 뒤 남은 마음은?
7. 지금 다시 돌아본다면 어떤 의미였나요?

### 검토 화면

저장 전에는 전체 답변과 대표 장면/감정을 정리한다.

```text
제주 여행

이 기억에서 가장 또렷한 장면은?
[바다 앞에서 먹은 라면]

이 기억에 가까운 감정은?
[설렘] [편안함] [아쉬움] [자유로움] [그리움] [기타]

내 답변
1. 그 여행에서 가장 먼저 떠오르는 장면은?
답변...

[수정하기]
[저장하기]
```

대표 장면/감정 선택은 필수 입력처럼 압박하지 않는다. 추천값을 보여주고 사용자가 바꿀 수 있게 한다.

## 11. 완료 화면

완료 화면은 기억이 저장되는 감각을 준다.

포함:

- 기억이 완성됐다는 문구
- 내 기억 공간에 저장되는 연출
- Google/Kakao 저장 CTA
- 임시 저장
- 임시 저장 데이터 유실 경고

구조:

```text
기억이 완성됐어요.

안전하게 보관하려면 로그인이 필요해요.
로그인하면 다른 기기에서도 다시 볼 수 있어요.

[Google로 저장하기]
[Kakao로 저장하기]

[임시로 저장하기]
브라우저 데이터가 삭제되면 임시 저장한 기억은 사라질 수 있어요.
```

저장 후에는 홈으로 이동하고, 새 기억 오브젝트가 생기는 장면을 보여준다.

## 12. 다시 보기

다시 보기는 사용자가 작성한 기억 묶음을 열고, 같은 질문의 답변을 비교하는 공간이다.

기본 구조:

```text
2026
[2026년의 나]
[2026 상반기의 나]
[2026 하반기의 나]

여행
[제주 여행의 기억]
```

상세 구조:

```text
제주 여행

자유로움

"바다 앞에서 먹은 라면"

그 여행에서 가장 먼저 떠오르는 장면은?
답변...

그때 가장 많이 느낀 감정은?
답변...

[같은 질문 비교]
[공유하기]
```

같은 질문 비교는 회고의 핵심 가치다.

```text
질문
가장 먼저 떠오르는 순간은?

2026 상반기
답변...

2026 하반기
답변...

2026년
답변...
```

## 13. 공유 화면

MVP 공유 화면은 읽기 전용이 목적이다.

친구가 할 수 있는 것:

- 공유된 회고 읽기
- 나도 기억 작성하기로 이동

친구가 할 수 없는 것:

- 원본 수정
- 삭제
- 댓글/좋아요
- 작성자 데이터 관리
- 같은 질문 답변 비교

공유 화면 문구:

```text
OO님의 기억을 보고 있어요

이 화면에서는 읽기만 할 수 있어요.
```

같은 질문 답변과 서로 답한 뒤 비교 공개는 MVP 이후 확장 기능으로 둔다.

## 14. 버튼

### 주요 버튼

사용처:

- 기억 작성 시작
- 다음 질문
- 저장
- Google로 저장하기
- Kakao로 저장하기
- 열어보기

규칙:

- 모바일 터치 높이 44px 이상
- 한 화면의 주요 CTA는 하나만 선명하게
- 기본 우주 테마에서는 `Memory Gold`를 중심으로 사용

### 보조 버튼

사용처:

- 임시로 저장하기
- 더 길게 쓰기
- 공유하기
- 가져오기/백업

규칙:

- 주요 CTA보다 낮은 대비
- 위치와 목적을 명확히
- 홈 장면을 방해하지 않음

### 위험 버튼

사용처:

- 기억 삭제
- 전체 데이터 삭제
- 로그아웃

규칙:

- 홈 첫 화면에 노출하지 않는다.
- 프로필 메뉴나 상세 하위 영역에 둔다.
- 확인 절차를 둔다.

## 15. 여백과 레이아웃

모바일 우선으로 설계한다.

| 역할 | 값 |
| --- | --- |
| 화면 좌우 padding | 18-20px |
| 섹션 간격 | 28-40px |
| 카드 내부 padding | 18-22px |
| 하단 탭 높이 | 64-76px |
| 터치 대상 최소 크기 | 44px |

규칙:

- 홈 장면은 충분한 숨 쉴 공간을 가진다.
- 질문 카드는 화면 중앙 집중을 유지한다.
- 하단 CTA는 키보드와 겹치지 않아야 한다.
- 모바일에서 텍스트가 버튼 안에서 줄바꿈/잘림되지 않아야 한다.

## 16. 애니메이션

허용:

- 별의 미세한 opacity 변화
- 오브젝트의 느린 부유
- 오브젝트 클릭 시 짧은 확대
- 질문 카드 좌우 슬라이드
- 완료 시 오브젝트가 공간에 놓이는 연출
- 미리보기 패널의 부드러운 등장

금지:

- 빠른 파티클 폭발
- 강한 네온 깜빡임
- 글쓰기 중 반복되는 큰 움직임
- 게임식 HUD
- 읽기를 방해하는 배경 애니메이션

권장 값:

```text
object float: 6-10s ease-in-out infinite
star twinkle: 3-6s ease-in-out infinite
card slide: 220-320ms ease
tap scale: 160-220ms ease-out
panel reveal: 240-320ms ease
screen transition: 240-360ms ease
```

## 17. 접근성

- 모든 주요 버튼은 키보드 포커스 상태를 가진다.
- 애니메이션은 `prefers-reduced-motion`을 따른다.
- 색상만으로 상태를 구분하지 않는다.
- 오브젝트 클릭 영역은 최소 44px 이상으로 만든다.
- 공유/읽기 화면은 읽기 순서가 자연스러워야 한다.
- 입력 화면에서는 질문, 입력창, 다음 버튼 순서가 명확해야 한다.
